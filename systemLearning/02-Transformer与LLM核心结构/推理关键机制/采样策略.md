# 采样策略

> **所属章节**: [[推理关键机制]]
> **所属模块**: [[02-Transformer与LLM核心结构]]
> **难度**: 中级

## 1. 一句话定义

**采样策略（sampling strategy，decoding strategy，采样策略/解码策略）** 是 LLM 在 [[autoregressive decoding]] 每一步、从 [[logits generation]] 得到的词表概率分布中**选出下一词**的方法；核心选项有 **greedy（贪心，argmax）**、**temperature（温度，调分布锐度）**、**top-k（截断到最高 k 个）**、**top-p / nucleus（累计概率截断）**，它们可组合使用，用来在**质量与多样性**之间权衡——greedy 确定但易重复啰嗦，采样类引入受控随机以提升多样性与自然度。

## 2. 为什么需要它（动机与背景）

[[logits generation]] 输出的是词表上的 logits，softmax 后是概率分布。如何"从分布里选一个词"看似简单，但选法影响巨大：

- **纯贪心（argmax）**：每步选概率最高词 → 确定输出，但易陷入重复循环、句子机械、缺乏多样性，长文尤甚。
- **纯随机（按概率抽）**：完全按分布抽 → 太随机，易选到低概率烂词，质量差、不可控。
- **受控采样**：在贪心与纯随机之间，用 temperature 调锐度、top-k/top-p 截断尾部烂词后再采样 → 既有多样性又保质量。

不同任务需求不同：代码/数学要确定（低温度、贪心），对话/创作要多样（高温度、top-p）。采样策略是调节 LLM 输出风格与可靠性的关键旋钮。

## 3. 核心概念详解

### 3.1 greedy（贪心）

每步取 $\arg\max$ 概率最高的词：

$$
\hat v = \arg\max_v p_v
$$

- 确定性：同输入永远同输出。
- 问题：易重复（"the the the..."）、易陷入局部最优循环、缺乏多样性。
- 适用：代码、数学、事实问答等需确定的场景。

### 3.2 temperature（温度）

采样前对 logits 除以 $T>0$ 再 softmax：

$$
p_v = \frac{\exp(z_v/T)}{\sum_k \exp(z_k/T)}
$$

- $T\to 0^+$：分布趋近 one-hot（趋 greedy），最确定。
- $T=1$：原始 softmax 分布。
- $T>1$：分布变平坦，更随机、更多样。
- $T<1$：分布变尖锐，更确定、更保守。
- **不改变 logits 排序**（只改分布的"陡峭度"），argmax 结果不变，但采样行为改变。

### 3.3 top-k

只保留 logits 最高的 **k 个**词，其余置 $-\infty$（概率清零），再在剩余 k 个里按（温度调整后的）概率采样：

$$
\text{mask } v \notin \text{top-}k,\quad p_v'=0;\quad \text{其余重归一化后采样}
$$

- 固定候选数 k（如 40/50）。
- 砍掉尾部烂词，保质量。
- 问题：k 固定不适应分布——平坦分布 k 太小丢好词，尖锐分布 k 太大留烂词。

### 3.4 top-p（nucleus，核采样）

保留**累计概率达到 p 的最小词集合**（核），其余清零，再采样：

1. 按概率降序排列。
2. 累计概率 $\sum$ 到 $\ge p$ 时停止，取这部分词（nucleus）。
3. 其余置 0，重归一化后采样。

- 候选数**自适应**：分布平坦时 nucleus 大（多候选），尖锐时 nucleus 小（少候选）。
- 比 top-k 更鲁棒，是当前主流（p=0.9/0.95 常用）。

### 3.5 组合与对比

实践中常组合：`top-k + top-p + temperature`。先 temperature 调锐度，再 top-k 截断，再 top-p 截断，最后在剩余里按概率采样。

| 策略 | 候选集 | 随机性 | 典型参数 | 适用 |
|---|---|---|---|---|
| greedy | argmax 1 个 | 无 | — | 代码/数学/事实 |
| temperature | 全词表 | 可调 | T=0.7~1.0 | 通用调锐度 |
| top-k | 固定 k 个 | 有 | k=40~50 | 通用（不适应分布） |
| top-p | 自适应 nucleus | 有 | p=0.9~0.95 | **主流**，对话/创作 |
| 组合 | k∩nucleus | 有 | T+top-k+top-p | 生产默认 |

## 4. 数学原理 / 公式

### temperature 的等价

$$
\text{softmax}(z/T)_v = \frac{\exp(z_v/T)}{\sum_k\exp(z_k/T)}
$$

$T\to0$：$\arg\max$ 不变但分布趋 one-hot；$T\to\infty$：趋均匀。温度等价于在 softmax 前对 logits 做整体缩放，改变相对概率比但不改排序。

### top-k / top-p 的截断+重归一

设截断后候选集 $\mathcal{S}$，则：

$$
p_v' = \begin{cases}\frac{p_v}{\sum_{u\in\mathcal{S}} p_u} & v\in\mathcal{S}\\ 0 & \text{否则}\end{cases}
$$

截断后必须**重归一化**使概率和为 1，再按 $p'$ 采样。

### top-p 的最小集合

$$
\mathcal{S}_p = \arg\min_{\mathcal{S}} |\mathcal{S}|\quad\text{s.t.}\quad \sum_{v\in\mathcal{S}} p_v \ge p
$$

按概率降序贪心加入直到累计 $\ge p$，得最小候选集。

### 采样的随机性

按 $p'$ 采样：$\hat v \sim \text{Categorical}(p')$。同样输入不同随机种子 → 不同输出（多样性来源）。greedy 是 $T\to0$ 或直接 argmax 的退化。

## 5. 代码示例

```python
import torch
import torch.nn.functional as F

def sample(logits, strategy='greedy', T=1.0, k=50, p=0.9):
    """从 logits (V,) 选一个 token id。"""
    if strategy == 'greedy':
        return logits.argmax()
    z = logits / T                              # temperature
    probs = F.softmax(z, dim=-1)
    # top-k
    if k > 0:
        topk_v, topk_i = probs.topk(k)
        mask = torch.full_like(probs, 0)
        mask[topk_i] = topk_v
        probs = mask / mask.sum()
    # top-p
    if p < 1.0:
        sorted_p, sorted_i = torch.sort(probs, descending=True)
        cum = sorted_p.cumsum(-1)
        keep = cum - sorted_p < p              # 累计达 p 前的保留
        sorted_p[~keep] = 0
        probs = torch.zeros_like(probs)
        probs[sorted_i] = sorted_p / sorted_p.sum()
    return torch.multinomial(probs, 1)

logits = torch.randn(32000) * 8
print('greedy:', sample(logits, 'greedy').item())
print('temp0.8+top-p0.9:', sample(logits, 'sample', T=0.8, k=0, p=0.9).item())

# HuggingFace 风格
# model.generate(do_sample=True, temperature=0.8, top_k=50, top_p=0.9,
#                max_new_tokens=100)
# model.generate(do_sample=False)               # greedy
```

## 6. 与其他知识点的关系

- **上游（依赖）**: [[logits generation]]（采样作用在 logits/概率上）、[[autoregressive decoding]]（每步采样一次）。
- **下游（应用）**: 推理服务（采样参数是 API 请求的核心配置）、创作/对话/代码等不同任务的参数调节。
- **对比 / 易混**:
  - **greedy vs beam search**：greedy 每步选最优（局部），beam search 保留多条候选路径（全局更优但慢，LLM 生成中已少用）。
  - **top-k vs top-p**：固定候选数 vs 自适应候选数，top-p 更鲁棒。
  - **temperature vs top-k/top-p**：温度调锐度（不改候选集），top-k/top-p 截断候选集（不改 logits 量级）。常组合。
  - **采样 vs greedy**：采样引入随机性（多样），greedy 确定（单调）。

## 7. 常见误区与易错点

> [!warning] 误区清单
> 1. **temperature 改变 logits 排序**：温度只缩放 logits，**不改变 argmax**，只改变采样分布的陡峭度。以为调温度能改变"最可能词"是错的。
> 2. **top-k 固定万能**：k 不适应分布，平坦/尖锐分布下同一 k 效果差异大，top-p 更鲁棒。
> 3. **截断后不重归一化**：top-k/top-p 砍掉词后概率和 $<1$，必须重归一化才能正确采样，否则概率失真。
> 4. **temperature=0 当作 greedy**：$T=0$ 数学上 undefined（除零），实现上要特殊处理为 argmax，不能真除以 0。
> 5. **以为温度越高越好**：高温度（>1.2）分布过平，易选烂词、输出乱，质量下降；温度要适度（0.7~1.0 常用）。
> 6. **greedy 一定最快**：greedy 省采样但易重复，长生成可能更长才停；且采样可复用 forward，速度差异主要在采样开销（小）。
> 7. **top-p 的 p 越大越多样**：p 大 = 候选多 = 更随机；p 小 = 候选少 = 更确定。搞反会设错参数。
> 8. **组合参数乱设**：top-k 与 top-p 同时开会取交集，要理解叠加效果；常见 temperature+top-p 即可，不必全开。

## 8. 延伸细节

### 8.1 min-p 采样

按"最高概率的 $\alpha$ 倍"为阈值截断（$p_v \ge \alpha\cdot p_{\max}$），自适应分布且更稳定，近来越来越流行，部分场景优于 top-p。

### 8.2 beam search

保留 top-B 条候选序列路径，结束时取总概率最高者。比 greedy 全局更优，但 LLM 生成中因计算贵、易重复、与指令微调目标不匹配，已基本被 top-p 采样取代，多用于翻译/摘要等封闭任务。

### 8.3 repetition penalty / presence / frequency

为防重复，对已出现词的 logits 乘惩罚因子降权。常与采样组合。LLM 生成默认常开轻度 repetition penalty（1.1~1.3）。

### 8.4 typical sampling / contrastive search

- **typical sampling**：选"信息熵接近平均"的词，避免过高或过低概率词，提升自然度。
- **contrastive search**：在采样的同时用对比排序降重复、提连贯，质量在多个基准上接近 greedy 又有多样性。

### 8.5 采样与指令微调

指令/RLHF 微调后的模型分布更尖锐（高置信），低温度（0.3~0.7）即可高质量输出；基座模型分布平坦，需更高温度或更强截断。采样参数要与模型类型匹配。

### 8.6 确定性与可复现

设随机种子 + 采样可复现；greedy 天然复现。生产中代码/事实任务常用 greedy 或低温度保确定，创意任务用采样保多样。API 常同时提供 `temperature`、`top_p`、`top_k` 供用户调。
