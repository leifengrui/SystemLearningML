# 自注意力

> **所属章节**: [[Transformer基础]]
> **所属模块**: [[02-Transformer与LLM核心结构]]
> **难度**: 中级

## 1. 一句话定义

**自注意力（self-attention，自注意力机制，self-attention）** 让序列中**每个位置**通过三组投影 **Query（Q）/Key（K）/Value（V）** 与**所有位置**（含自己）计算相关性，按相关性加权求和得到该位置的新表示；核心公式 $\text{Attention}(Q,K,V)=\text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$，一步捕捉任意距离依赖、且全并行。

## 2. 为什么需要它（动机与背景）

序列建模的老办法（RNN/LSTM）痛点：

- **串行**：要按时间步一个个算，$n$ 长序列要 $O(n)$ 步，GPU 吃不饱、长序列慢。
- **长距离衰减**：信息要经多次隐状态传递，远距离依赖容易丢。
- **无法并行**：训练慢。

自注意力（Transformer 2017）换思路：**每个位置直接看所有位置**，用"加权求和"一次性聚合全局信息。距离不再是障碍（一步到位），且全是矩阵乘法 → 高度并行、GPU 友好。代价是 $O(n^2)$ 复杂度（见 8.1）。

## 3. 核心概念详解

### 3.1 Q / K / V 三元组（类比检索）

借用信息检索的类比：

- **Query（查询）**：当前位置"我想找什么"。
- **Key（键）**：每个位置"我能被什么找到"（被检索的标签）。
- **Value（值）**：每个位置"我实际承载的内容"。

注意力 = 用 Q 和每个 K 算相似度 → softmax 成权重 → 用权重对 V 加权求和。哪个位置的 K 与当前 Q 最匹配，就多取它的 V。

### 3.2 计算流程（逐行）

输入 $X\in\mathbb{R}^{n\times d}$（$n$ 个 token，$d$ 维），投影：

$$
Q=XW_Q,\quad K=XW_K,\quad V=XW_V\qquad (W_*\in\mathbb{R}^{d\times d_k})
$$

打分：$S=QK^\top\in\mathbb{R}^{n\times n}$，$S_{ij}$=位置 $i$ 的 Q 与位置 $j$ 的 K 的点积（相似度）。

缩放 + softmax：$A=\text{softmax}\!\left(S/\sqrt{d_k}\right)$，每行归一化成权重。

输出：$O=AV\in\mathbb{R}^{n\times d_v}$，每个位置是 V 的加权和。

> [!note] 一次矩阵乘法搞定全序列
> 整个过程就是几次 matmul + softmax，没有时间步循环 → 全并行，这是 Transformer 速度的根源。

### 3.3 为什么除以 $\sqrt{d_k}$（scaled dot-product）

点积 $q\cdot k=\sum_{i=1}^{d_k}q_i k_i$，若 $q_i,k_i$ 独立均值 0 方差 1，则 $q\cdot k$ 方差为 $d_k$、标准差 $\sqrt{d_k}$。$d_k$ 大时点积量级大 → softmax 进入**饱和区**（梯度趋 0、one-hot 化）→ 训练不动。除以 $\sqrt{d_k}$ 把方差拉回 1，让 softmax 在有梯度的区间工作。这是 "scaled" 的全部含义。

### 3.4 三种注意力类型

| 类型 | Q 来自 | K,V 来自 | 用途 |
|---|---|---|---|
| **self-attention**（自） | 同序列 | 同序列 | Encoder（双向，看全部） |
| **causal self-attention**（单向） | 同序列 | 同序列 + [[causal mask]] | Decoder（只看过去） |
| **cross-attention**（交叉） | decoder | encoder 输出 | 翻译/多模态 |

LLM（decoder-only）用的是**因果自注意力**：Q/K/V 都来自同一序列，但用 causal mask 遮住未来位置。

### 3.5 双向 vs 单向

- **双向**（BERT）：每个位置看全序列（左右都看），适合理解任务。
- **单向/因果**（GPT）：每个位置只看自己和之前的位置，适合生成（不能偷看未来 token）。

> [!warning] 生成必须因果
> 自回归生成时，当前位置不能看到未来的 token（否则训练-推理不一致、信息泄露）。用 [[causal mask]] 把上三角置 $-\infty$，softmax 后权重为 0。

## 4. 数学原理 / 公式

### 核心公式

$$
\text{Attention}(Q,K,V)=\text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
$$

展开每个位置 $i$：

$$
\text{out}_i=\sum_{j=1}^{n}\underbrace{\frac{\exp(q_i^\top k_j/\sqrt{d_k})}{\sum_{l}\exp(q_i^\top k_l/\sqrt{d_k})}}_{\text{注意力权重 }a_{ij}}\,v_j
$$

### softmax 的行归一化

$A$ 的每行（每个 query 位置）对所有 key 归一化 → 权重和为 1 → 输出是 V 的凸组合。

### 梯度视角

反向时，softmax + matmul 的梯度让每个位置都对相关位置有梯度路径 → 一步传遍全序列，无距离衰减（与 RNN 的远距离梯度消失对比）。

## 5. 代码示例

```python
import torch
import torch.nn.functional as F

# === 朴素实现（教学用）===
def attention_naive(Q, K, V, mask=None):
    # Q,K,V: (n, d_k)
    d_k = Q.size(-1)
    scores = Q @ K.transpose(-2,-1) / (d_k ** 0.5)   # (n,n) 相似度，缩放
    if mask is not None:
        scores = scores.masked_fill(mask == 0, float('-inf'))  # causal/padding
    A = F.softmax(scores, dim=-1)                     # 每行归一化
    return A @ V, A                                    # 返回输出和权重(可可视化)

n, d = 5, 64
Q = torch.randn(n, d); K = torch.randn(n, d); V = torch.randn(n, d)
out, weights = attention_naive(Q, K, V)
print(out.shape, weights.shape)                      # (5,64) (5,5)

# causal mask（下三角为 1）
causal = torch.tril(torch.ones(n, n)).bool()
out_c, w_c = attention_naive(Q, K, V, mask=causal)
print((w_c[0, 1:] == 0).all())                        # 位置 0 看不到未来 → True

# === 工程实现：用 fused kernel（快、省显存，含 FlashAttention）===
# 推荐用 F.scaled_dot_product_attention，内部自动选最优 kernel
attn = F.scaled_dot_product_attention(Q, K, V, is_causal=True)
# 等价于上面的 causal 自注意力，但快几个量级、显存 O(n) 而非 O(n²)
```

## 6. 与其他知识点的关系

- **上游（依赖）**: [[Tensor]]（矩阵运算）、[[损失函数分类]]（attention 输出接 loss）。
- **下游（应用）**: [[多头注意力]]（实际用的是多头版）、[[KV cache]]（推理时缓存 K/V）、[[causal mask]]、[[位置编码]]（attention 本身无序，要加位置信息）、[[flash attention]]。
- **对比 / 易混**:
  - **self vs cross attention**：Q/K/V 是否同源。self 同源、cross 跨序列。
  - **attention vs RNN/CNN**：attention 全并行+无距离衰减但 $O(n^2)$；RNN 串行 $O(n)$；CNN 局部并行、感受野靠堆叠。
  - **self-attention vs [[多头注意力]]**：单头是基础单元，多头是并行多组、捕捉不同模式。

## 7. 常见误区与易错点

> [!warning] 误区清单
> 1. **忘除 $\sqrt{d_k}$**：$d_k$ 大时 softmax 饱和、梯度消失，训练不动。
> 2. **causal mask 用错符号**：要 mask 的位置填 $-\infty$（softmax 后为 0），不是填 0（softmax(0) 仍非 0）。
> 3. **以为 attention 有顺序感知**：纯 attention 对输入顺序不变（置换等变），必须配 [[位置编码]] 才区分顺序。
> 4. **softmax 维度搞错**：按最后一维（key 维）归一化，不是 query 维。
> 5. **生成时不 mask 未来**：训练时 mask 了，推理若不 mask 会信息泄露、与训练分布不一致。
> 6. **以为 $O(n^2)$ 只是显存**：计算量、显存、注意力权重矩阵都是 $O(n^2)$，长序列瓶颈。
> 7. **Q/K/V 用同一矩阵**：必须三个不同投影 $W_Q,W_K,W_V$，否则退化。

## 8. 延伸细节

### 8.1 $O(n^2)$ 瓶颈与 FlashAttention

注意力分数矩阵 $n\times n$，长序列（LLM 8k~128k）显存/计算爆炸。[[flash attention]] 用 tiling 不显式存 $n\times n$ 矩阵，把显存降到 $O(n)$、计算仍 $O(n^2)$ 但 IO 大减，是现代 LLM 必备。

### 8.2 线性注意力

用核函数近似 $\text{softmax}(QK^\top)$，把 $O(n^2)$ 降到 $O(n)$（Linformer/Performers），但质量损失，未主流。

### 8.3 注意力可解释性

注意力权重常被当作"模型在看哪"，但权重高不等于因果重要（可被其他头补偿），严谨归因要用 integrated gradients / attention rollout，别过度解读。

### 8.4 attention sink

长序列生成时，若干位置（常是首 token/BOS）注意力异常高，像"注意力 sink"。研究认为模型用它卸载不必要注意力，相关 KV 量化压缩要保护这些 sink。

### 8.5 与位置编码的耦合

[[RoPE]] 直接作用在 Q/K 上（旋转），让相对位置融入点积；[[sinusoidal PE]] 加在输入上。不同位置编码与 attention 的耦合方式不同，影响外推性。
