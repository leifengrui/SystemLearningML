# 多头注意力

> **所属章节**: [[Transformer基础]]
> **所属模块**: [[02-Transformer与LLM核心结构]]
> **难度**: 中级

## 1. 一句话定义

**多头注意力（multi-head attention，多头注意力，MHA）** 把 Q/K/V 沿特征维拆成 $h$ 组（"头"），每组独立做 [[自注意力]]，再把结果拼接投影回原维；不同头关注**不同子空间/模式**（有的看语法、有的看位置、有的看长程），是 Transformer 实际使用的注意力形态。

## 2. 为什么需要它（动机与背景）

单头 [[自注意力]] 的问题：

- 一个头只能学**一种**注意力模式（一种 Q-K 匹配方式）。
- 表达力受 $d_k$ 限制，想同时捕捉"局部语法"+"长程依赖"+"位置关系"力不从心。
- 直接增大 $d_k$ 会让 softmax 更易饱和、参数贵。

**多头**的思路：与其用一个大的注意力，不如用 $h$ 个小的并行注意力，每个在低维子空间学不同模式，最后融合。类比"多个专家从不同角度看同一段话"。代价几乎只是多一组投影参数，计算量与单头（同总维度）相当。

## 3. 核心概念详解

### 3.1 分头与拼接

输入 $X\in\mathbb{R}^{n\times d}$，先投影到 $Q,K,V$（维度 $d$），再**沿特征维切成 $h$ 段**，每段 $d_k=d/h$：

$$
Q=[Q_1;\dots;Q_h],\quad K=[K_1;\dots;K_h],\quad V=[V_1;\dots;V_h]
$$

每个头独立算注意力：

$$
\text{head}_i=\text{Attention}(QW_i^Q,\,KW_i^K,\,VW_i^V)\in\mathbb{R}^{n\times d_k}
$$

拼接后投影回 $d$ 维：

$$
\text{MHA}(X)=[\text{head}_1;\dots;\text{head}_h]\,W^O,\quad W^O\in\mathbb{R}^{d\times d}
$$

### 3.2 维度关系（关键）

- 模型宽 $d$、头数 $h$、每头宽 $d_k=d/h$。
- **总计算量与单头（同 $d$）相近**：$h$ 个小头并行 ≈ 1 个大头。
- **参数量**：$4d^2$（$W_Q,W_K,W_V,W^O$ 各 $d^2$），与头数无关（只是切法不同）。

> [!note] 头数不增计算量
> 把 $d$ 切成 $h$ 个 $d/h$，总 matmul 量不变。头数改变的是"子空间数"，不是计算量。所以加头几乎免费（除非改 $d_k$）。

### 3.3 头数与 head_dim

- 头数 $h$：8、16、32、...（Llama-7B 用 32 头）。
- head_dim $d_k$：常 64、128。太小（<32）表达力差，太大 softmax 易饱和。
- 经验：$d_k\approx64$~$128$ 较稳。

### 3.4 MHA vs MQA vs GQA（推理显存关键）

| 变体 | Q 头数 | K/V 头数 | 显存(KV cache) | 质量 | 用例 |
|---|---|---|---|---|---|
| **MHA** | $h$ | $h$ | 大 | 最好 | GPT-2/BERT |
| **MQA** | $h$ | **1** | **最小** | 略降 | 早期推理优化 |
| **GQA** | $h$ | $g$（$1<g<h$） | 中 | 接近 MHA | **Llama2-70B/Llama3** |

- **MQA**（Multi-Query Attention）：所有 Q 头共享 1 组 K/V，[[KV cache]] 省到 $1/h$，但质量掉。
- **GQA**（Grouped-Query Attention）：折中，Q 头分成 $g$ 组，每组共享 1 组 K/V。Llama2-70B 用 8 组（$h=64$）、Llama3 延续。质量接近 MHA、显存大幅省。

> [!tip] 为什么 LLM 都转 GQA
> 推理时 KV cache 显存是长上下文瓶颈（见 [[KV cache]]）。GQA 把 K/V 头数从 $h$ 降到 $g$，KV cache 省到 $g/h$，质量损失极小，是 Llama 系及后续 LLM 的标准选择。

### 3.5 头的特化现象

训练后观察到头有分工：

- **语法头**：关注句法依赖（主谓、修饰）。
- **位置头**：关注相邻/固定相对位置。
- **长程头**：关注远距离实体。
- **少用头**：可剪枝（attention pruning）。

## 4. 数学原理 / 公式

### 多头拼接的等价性

把 $h$ 个头的 $W_i^*$ 拼成块对角矩阵，多头注意力等价于"在 $d$ 维做一次注意力但 Q/K/V 用块对角投影"。所以多头 ≈ 单头但**强制不同子空间独立**，是一种结构先验，让优化更易学到多样模式。

### softmax 在低维更稳

每个头 $d_k=d/h$ 较小，点积方差 $d_k$ 小，softmax 不易饱和 → 除 $\sqrt{d_k}$ 后梯度好。这也是多头比单头（大 $d_k$）好训的原因之一。

## 5. 代码示例

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class MHA(nn.Module):
    def __init__(self, d, h):
        super().__init__()
        assert d % h == 0
        self.h, self.dk = h, d // h
        self.Wq = nn.Linear(d, d, bias=False)
        self.Wk = nn.Linear(d, d, bias=False)
        self.Wv = nn.Linear(d, d, bias=False)
        self.Wo = nn.Linear(d, d, bias=False)
    def forward(self, x, causal=False):
        B, n, d = x.shape
        q = self.Wq(x).view(B, n, self.h, self.dk).transpose(1, 2)  # (B,h,n,dk)
        k = self.Wk(x).view(B, n, self.h, self.dk).transpose(1, 2)
        v = self.Wv(x).view(B, n, self.h, self.dk).transpose(1, 2)
        # 每头独立 attention（fused kernel）
        o = F.scaled_dot_product_attention(q, k, v, is_causal=causal)  # (B,h,n,dk)
        o = o.transpose(1, 2).reshape(B, n, d)        # 拼接回 (B,n,d)
        return self.Wo(o)

mha = MHA(d=512, h=8)
x = torch.randn(2, 16, 512)
print(mha(x, causal=True).shape)                     # (2,16,512)

# 工程实现直接用 nn.MultiheadAttention 或 SDPA，别手写
# === GQA 示例（Llama 风格）：Q 头 h，KV 头 g ===
class GQA(nn.Module):
    def __init__(self, d, h, g):
        super().__init__()
        assert d % h == 0 and h % g == 0
        self.h, self.g, self.dk = h, g, d // h
        self.Wq = nn.Linear(d, h*self.dk, bias=False)
        self.Wk = nn.Linear(d, g*self.dk, bias=False)   # K/V 头数=g
        self.Wv = nn.Linear(d, g*self.dk, bias=False)
        self.Wo = nn.Linear(d, d, bias=False)
    def forward(self, x, causal=True):
        B, n, d = x.shape
        q = self.Wq(x).view(B,n,self.h,self.dk).transpose(1,2)   # (B,h,n,dk)
        k = self.Wk(x).view(B,n,self.g,self.dk).transpose(1,2)   # (B,g,n,dk)
        v = self.Wv(x).view(B,n,self.g,self.dk).transpose(1,2)
        # 把 g 组 K/V 重复到 h 组（每 g 个 Q 头共享一组 KV）
        rep = self.h // self.g
        k = k.repeat_interleave(rep, dim=1)            # (B,h,n,dk)
        v = v.repeat_interleave(rep, dim=1)
        o = F.scaled_dot_product_attention(q,k,v, is_causal=causal)
        return self.Wo(o.transpose(1,2).reshape(B,n,d))

gqa = GQA(d=512, h=16, g=4)                           # 16 Q 头，4 KV 组
print(gqa(x).shape)                                   # (2,16,512)  KV cache 省 4x
```

## 6. 与其他知识点的关系

- **上游（依赖）**: [[自注意力]]（多头 = 多个并行单头）。
- **下游（应用）**: [[decoder-only architecture]]（每层一个 MHA/GQA）、[[KV cache]]（KV 头数决定 cache 大小，GQA/MQA 省）、[[flash attention]]（融合多头的 kernel）。
- **对比 / 易混**:
  - **MHA vs MQA vs GQA**：见 3.4，KV 头数不同，影响推理显存。
  - **多头 vs 单头**：同总 $d$ 下计算量相当，多头表达力更强、更易训。
  - **头 vs 层**：头是同一层内并行，层是串行堆叠。

## 7. 常见误区与易错点

> [!warning] 误区清单
> 1. **以为头数越多越好**：$d_k$ 太小（<32）表达力差，且头会冗余。有最优点。
> 2. **GQA 的 $g$ 设 1 当 MQA**：$g=1$ 就是 MQA，质量掉；$g=h$ 退回 MHA。中间折中。
> 3. **K/V 头数与 Q 不匹配时忘 repeat**：GQA 要把 KV 头扩展到与 Q 头对齐才能算。
> 4. **view/transpose 维度错**：分头是 `view(B,n,h,dk).transpose(1,2)` 成 `(B,h,n,dk)`，弄反成 `(B,n,h,dk)` 会让头混进序列维。
> 5. **以为多头能自动分工**：分工是训练后涌现的，不是结构保证；有些头会冗余可剪。
> 6. **推理 KV cache 按 $h$ 算**：GQA/MQA 应按 KV 头数 $g$ 算 cache，否则高估 $h/g$ 倍。

## 8. 延伸细节

### 8.1 头剪枝

训练后部分头冗余（注意力近似 identity），可安全移除减推理成本。但哪些头关键要分析（如 attention sink 头不能剪）。

### 8.2 head_dim 的选择

小模型 $d_k=64$，大模型（Llama-70B）$d_k=128$。$d_k$ 大表达力强但 softmax 易饱和、显存涨。需与 $h$ 平衡。

### 8.3 MQA/GQA 的训练稳定性

KV 头减少让梯度路径变少，训练初期略不稳，需配 [[学习率调度]] warmup。Llama 用 GQA + warmup 稳定。

### 8.4 与 RoPE 的配合

[[RoPE]] 作用在每头的 Q/K 上（按 head_dim 旋转），GQA 下只对 Q 和 K 头分别施旋转，V 不旋转。

### 8.5 多头注意力与多模态

cross-attention 的多头版用于多模态（Q 来自文本、K/V 来自图像 patch），每个头学一种跨模态对齐模式，是 BLIP/LLaVA 等的结构。
