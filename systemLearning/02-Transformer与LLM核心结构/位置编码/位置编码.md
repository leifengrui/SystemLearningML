# 位置编码

> **所属章节**: [[位置编码]]
> **所属模块**: [[02-Transformer与LLM核心结构]]
> **难度**: 中级

## 1. 一句话定义

**位置编码（Position Encoding / Position Embedding，PE，位置编码/位置嵌入）** 是给本身**无序**的 [[自注意力]] 注入位置信息的方法统称：按"每个位置一个向量加到输入"（**绝对位置编码**）或"在 attention 里编码 token 间相对距离"（**相对位置编码**）两种范式实现，代表方案分别是 [[sinusoidal PE]] 与 [[RoPE]]；它的核心使命是让模型能区分"猫追狗"与"狗追猫"这类**顺序敏感**的语义，并尽量支持**长度外推**（训练短、推理长）。

## 2. 为什么需要它（动机与背景）

[[自注意力]] 的本质是"对每个 token，用 Q 去和所有位置的 K 做加权求和"。这个操作**对输入顺序完全不变**（置换等变，permutation-equivariant）：

- 把输入 token 打乱顺序，attention 的输出（按相同置换）打乱，但**每个 token 看到的"全局"内容集合不变**。
- 等价于"词袋模型"：模型只知道"这批词有哪些"，不知道"谁在前谁在后"。
- 但语言顺序承载语义："猫追狗" ≠ "狗追猫"、"我帮你" ≠ "你帮我"。

所以**必须额外**把位置信息注入模型。这就是位置编码的职责。没有它，Transformer 就是个强大的词袋模型，连基本语序都分不清。

> [!note] attention 为什么无序
> attention 的核心 $\text{softmax}(QK^T/\sqrt d)V$ 里，Q、K、V 是对每个 token **独立**用同一组权重投影出来的。打乱 token 顺序，Q/K/V 按相同顺序打乱，softmax 加权求和的结果也按同序打乱——输出是输入置换的同步置换，**模型内部没有任何环节读取"第几个位置"**。所以 attention 对顺序无感，必须靠位置编码补上。

## 3. 核心概念详解

### 3.1 绝对位置编码（absolute PE）

给每个**绝对位置** $t=0,1,\dots,L-1$ 一个向量 $p_t\in\mathbb{R}^d$，加到该位置 token 的嵌入上：

$$
\tilde x_t = x_t + p_t
$$

- $p_t$ 可以是**固定公式**（[[sinusoidal PE]]，sin/cos 构造）或**可学习**（learned PE，一个 $L\times d$ 查表）。
- 注入点在**输入嵌入层**，对 attention/FFN 都透明。
- 局限：每个位置独立编码，**不直接表达"token 间距离"**；learned PE 无法外推到训练时没见过的更长位置。

### 3.2 相对位置编码（relative PE）

不编码"第几个位置"，而编码"token $i$ 和 token $j$ 之间差几步 $j-i$"，把它注入 attention 的 $QK^T$（或加偏置）：

$$
\text{attn}_{ij} = q_i k_j^T + b_{j-i}\quad\text{或}\quad q_i^T (k_j + \tilde r_{j-i})
$$

- 直接建模相对距离，符合"语言里重要的是谁离谁多远"。
- 外推性更好（相对距离的取值范围有限，长序列也只产生训练见过的差值）。
- 代表：T5 relative bias、[[RoPE]]（旋转式）、ALiBi（线性偏置）。

### 3.3 三种注入方式对比

| 方式 | 注入点 | 形式 | 代表 |
|---|---|---|---|
| **加法式** | 输入嵌入 | $\tilde x_t=x_t+p_t$ | sinusoidal PE、learned PE |
| **旋转式** | Q/K（每头） | $q_t\to R_t q_t$ | [[RoPE]] |
| **偏置式** | attention logits | $QK^T \to QK^T + b_{j-i}$ | T5 bias、ALiBi |

> [!tip] 旋转式 vs 加法式
> 加法式把位置"粘"在输入上，attention 要通过学习才能解出位置关系，耦合松散、外推差。旋转式**直接作用在 Q/K 上**，通过旋转矩阵的数学性质让 $q_i\cdot k_j$ 天然只依赖 $j-i$，耦合紧、外推好，成为现代 LLM 主流。

### 3.4 长度外推（extrapolation）

训练长度 $L_{\text{train}}$，推理长度 $L_{\text{test}}>L_{\text{train}}$ 时，模型能否保持质量？这是位置编码的分水岭：

- **learned PE**：查表只到 $L_{\text{train}}$，超了没向量 → **不能外推**。
- **[[sinusoidal PE]]**：公式可生成任意位置，理论可外推；但实测长序列质量衰减（attention 没学好远距离关系）。
- **[[RoPE]]**：相对位置，配合缩放/插值可外推 → 主流。
- **ALiBi**：偏置是距离的简单线性函数，天然外推，适合长序列。

外推改进方案见各笔记的延伸节（Position Interpolation、YaRN、NTK-aware）。

## 4. 数学原理 / 公式

### 4.1 attention 的置换等变性

设输入序列 $X=[x_1,\dots,x_L]^T$，置换 $\pi$ 作用得 $\pi X$。attention 层 $f(X)=\text{softmax}(XW_Q(XW_K)^T/\sqrt d)XW_V$。因 $W_Q,W_K,W_V$ 行独立作用于每个 token：

$$
f(\pi X)=\pi f(X)
$$

即输出同步置换 → 模型不区分顺序 → **需要位置编码**。

### 4.2 绝对 PE 的注入

$$
\tilde X = XW_e + P
$$

$P\in\mathbb{R}^{L\times d}$ 为位置矩阵（sinusoidal 公式或 learned 查表）。

### 4.3 相对 PE 在 attention

$$
\text{score}_{ij} = q_i k_j^T + b_{j-i}
$$

$b_{j-i}$ 只依赖相对距离 → 天然处理"多远"，与绝对位置无关。

### 4.4 RoPE 的相对性（核心）

[[RoPE]] 对位置 $t$ 的 Q 作用旋转 $R_t$：

$$
q_t' = R_t q_t,\quad k_j'=R_j k_j
$$

因旋转矩阵性质 $R_t^T R_j=R_{j-t}$：

$$
\langle q_t', k_j'\rangle = q_t^T R_t^T R_j k_j = q_t^T R_{j-t} k_j
$$

**内积只依赖相对位置 $j-t$** → 这就是 RoPE "相对位置编码"的数学根源，详见 [[RoPE]]。

## 5. 代码示例

```python
import torch
import torch.nn as nn

# 通用：绝对PE（加法式）占位——具体实现在 [[sinusoidal PE]]
class AbsPE(nn.Module):
    def __init__(self, d, L):
        super().__init__()
        # 这里用 learned PE 占位；sinusoidal 见专门笔记
        self.pe = nn.Parameter(torch.zeros(L, d))
    def forward(self, x):                # x: (B, n, d)
        return x + self.pe[:x.size(1)]

# 通用：相对偏置式 PE（T5 风格，示意）
class RelBias(nn.Module):
    def __init__(self, h, max_rel=128):
        super().__init__()
        # 把相对距离 j-i 映射到桶，查表得偏置
        self.bias = nn.Parameter(torch.zeros(h, max_rel*2+1))
    def forward(self, scores):           # scores: (B, h, n, n)
        n = scores.size(-1)
        idx = torch.arange(n)
        rel = (idx[None,:] - idx[:,None]).clamp(-64,64) + 64
        return scores + self.bias[:, rel]   # 广播到 batch

# 真正主流的 RoPE 见 [[RoPE]]，这里只示意接口
# from somewhere import apply_rope
# q, k = apply_rope(q, k, freqs)  # 旋转作用在 Q/K，V 不动
```

## 6. 与其他知识点的关系

- **上游（依赖）**: [[自注意力]]（PE 是为弥补 attention 的无序性而生）、[[多头注意力]]（旋转/偏置式作用在每头）、[[Tensor]]。
- **下游（应用）**: [[decoder-only architecture]]（每层用 RoPE）、长度外推工程（PI/YaRN）、长文本生成。
- **对比 / 易混**:
  - **绝对 PE vs 相对 PE**：见 3.1/3.2，注入点与外推性不同。
  - **[[sinusoidal PE]] vs learned PE**：固定公式 vs 查表，外推性差异。
  - **[[RoPE]] vs ALiBi**：旋转式 vs 偏置式，都是相对，但实现与特性不同。
  - **位置编码 vs token embedding**：后者编码"是什么词"，前者编码"在第几位"。

## 7. 常见误区与易错点

> [!warning] 误区清单
> 1. **以为 attention 自带顺序感知**：纯 attention 是置换等变，必须配 PE 才有序。这是最根本的误区。
> 2. **把绝对 PE 和相对 PE 混为一谈**：绝对编码"第几位"，相对编码"差几步"，外推性差异大。
> 3. **以为 learned PE 能外推**：查表只到训练长度，超位置无向量，硬外推会崩。
> 4. **以为 RoPE 加在输入**：RoPE 作用在**每头的 Q/K**（旋转），V 和输入嵌入都不动。
> 5. **忽视注入点差异**：加法式（输入）、旋转式（Q/K）、偏置式（logits）三者不可互换，混用会破坏位置语义。
> 6. **以为任意 sin/cos 组合都是 sinusoidal PE**：频率递减、奇偶配对、基数 10000 都有讲究，乱写外推性会丢（见 [[sinusoidal PE]]）。

## 8. 延伸细节

### 8.1 ALiBi（线性偏置）

不学位置编码，直接在 attention logits 上加一个与距离成正比的负偏置 $-m\cdot(j-i)$（$m$ 按头不同）。天然外推、无参数，适合极长序列。与 [[RoPE]] 是长序列方案的两大主流。

### 8.2 NoPE（无位置编码）

研究发现 decoder-only 因果 mask 本身隐含了"位置 $i$ 只看 $j\le i$"的偏序信息，某些任务不加 PE 也能学到顺序。但主流仍加 RoPE 以稳。

### 8.3 长度外推工程

训练 2k/4k、推理 32k/128k 的需求催生一批方法：**Position Interpolation（PI，线性压缩位置）**、**NTK-aware scaling（按频率分层缩放）**、**YaRN（混合插值）**，核心都是改造 [[RoPE]] 的频率/角度使其在更长位置仍稳定。详见 [[RoPE]] 延伸节。

### 8.4 位置编码的选择趋势

- BERT/原版 Transformer：sinusoidal 或 learned 绝对 PE。
- T5：相对偏置。
- 现代 LLM（Llama/Qwen/Mistral）：**RoPE**（事实标准）。
- 长序列特化：ALiBi 或 RoPE + 外推工程。
