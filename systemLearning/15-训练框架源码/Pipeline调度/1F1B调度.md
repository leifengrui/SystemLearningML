# 1F1B调度

> **所属章节**: [[Pipeline调度]]
> **所属模块**: [[15-训练框架源码]]
> **别名**: 1F1B / One Forward One Backward / 1F1B 调度 / 流水线 1F1B / GPipe 改进调度 / warmup-steady-cooldown
> **难度**: 中高（需懂 [[Pipeline Parallel]] + [[forward与backward]] + [[micro-batching]]）


## 1. 一句话定义

**1F1B 调度（One Forward One Backward）** 是 [[Megatron-LM]] 对朴素 [[Pipeline Parallel]] 的 GPipe 调度的改进——GPipe 先做**全部** $M$ 个 micro-batch 的 forward 再做全部 backward，导致 stage 的激活要存 $M$ 份（激活 $\propto M$，大）；**1F1B** 改为"做够 warmup 个 forward 后，进入 steady state 每做 1 个 forward 立刻跟 1 个 backward"（forward 与 backward 交错），让每 stage 最多只需缓存 $\approx P$（stage 数）个 micro-batch 的激活（vs GPipe 的 $M$ 个），**activation memory 降 $M/P$ 倍**；气泡（bubble）比例仍为 $\frac{P-1}{M+P-1}$（与 GPipe 同），但显存大省。调度分三阶段：**warmup**（stage $i$ 做 $P-1-i$ 个 forward 填满流水线）、**steady state**（1F1B 交替，forward 与 backward 交错）、**cooldown**（做完剩余 backward 排空流水线）。是 Megatron PP 的默认调度（[[Pipeline Parallel]] 的标准实现），[[Interleaved schedule]] 是其进一步减气泡的演进。

> [!note] 三句话定位
> - **是什么**：GPipe 改进——warmup 填流水线→steady 1F1B 交错→cooldown 排空，每 stage 只缓存 $\approx P$ 个 micro-batch 激活（vs GPipe 的 $M$）。
> - **为什么**：GPipe 存 $M$ 份激活显存爆；1F1B 用 forward/backward 交错让激活早 backward 早释放，省 $M/P$ 倍显存。
> - **与 [[Interleaved schedule]] 关系**：1F1B 气泡 $\frac{P-1}{M+P-1}$，interleaved（virtual stage）把气泡再除 $V$（virtual stage 数），是 1F1B 的演进。


## 2. 为什么需要它（动机与背景）

### 2.1 GPipe 的激活显存爆

[[Pipeline Parallel]] 的 GPipe 调度：stage $i$ 把所有 $M$ 个 micro-batch 的 forward 都做完，再做 backward。每 forward 算的激活要存到 backward 用。stage $i$ 存 $M - i$ 个 micro-batch 的激活（越早的 stage 存越多）。$M$ 大（流水线效率高需大 $M$）时激活显存爆（$\propto M$）。是 GPipe 的痛点。

### 2.2 forward/backward 交错省激活

1F1B：做够 warmup forward（填满流水线）后，每做 1 forward 立刻 1 backward。backward 释放该 micro-batch 的激活。所以每 stage 最多缓存 $\approx P$ 个（流水线深度，warmup 数）而非 $M$ 个。激活 $\propto P$（vs GPipe $\propto M$）。$M \gg P$ 时省显著。

### 2.3 气泡不变

GPipe 与 1F1B 的气泡比例都是 $\frac{P-1}{M+P-1}$（warmup $P-1$ 个 micro-batch 时间填 + cooldown 排空）。1F1B 不减气泡，只省显存。减气泡靠 [[Interleaved schedule]]（virtual stage）。

### 2.4 三阶段调度

warmup（填）→ steady（1F1B）→ cooldown（排空）。三阶段控制 forward/backward 的交错时序。是 Megatron `schedules.py` 的核心。

### 2.5 通信与计算交错

1F1B 的 forward/backward 间有 `send/recv`（micro-batch 跨 stage）。send/recv async 与计算 overlap（[[overlap strategy]]）。是 PP 的通信。


## 3. 核心概念详解

### 3.1 三阶段调度

```
设 P=4 stage, M=8 micro-batch.

GPipe (全 forward 再全 backward):
  stage 0: F0 F1 F2 F3 F4 F5 F6 F7 | B0 B1 B2 B3 B4 B5 B6 B7
  stage 1:    F0 F1 ... F7         |    B0 ... B7
  ...
  激活: stage 0 存 8 份 (M=8)

1F1B:
  stage 0: F0 F1 F2 F3 | F4 B0 F5 B1 F6 B2 F7 B3 | B4 B5 B6 B7
                     ^warmup       ^steady(1F1B)        ^cooldown
  warmup: P-1-i=3 个 forward (stage 0 填满)
  steady: 1F1B 交错 (F4-B0-F5-B1-...)
  cooldown: 剩余 backward (B4-B5-B6-B7)
  激活: stage 0 最多缓存 P=4 份 (warmup 数)
```

### 3.2 warmup 阶段

每 stage $i$ 做 $P-1-i$ 个 forward（stage 0 做 $P-1$ 个，stage $P-1$ 做 0 个）。填满流水线（每 stage 有足够 micro-batch 等下流 backward）。warmup 数 = stage 的"流水线深度"（需缓存的激活数）。

### 3.3 steady state（1F1B）

warmup 后进入 steady：每 stage 交替做 1 forward + 1 backward。backward 释放该 micro-batch 的激活（最早 forward 的）。forward 算新 micro-batch（激活进缓存）。缓存数稳定在 $\approx P$（一进一出）。

### 3.4 cooldown 阶段

forward 做完后（$M$ 个 forward 完成），cooldown 做剩余 backward（排空流水线）。cooldown 数 = warmup 数（对称）。做完所有 backward。

### 3.5 激活显存

GPipe：stage $i$ 存 $M - i$ 份激活（$\propto M$）。1F1B：每 stage 存 $\approx P$ 份（warmup 数，$\propto P$）。$M \gg P$ 时省 $M/P$ 倍。典型 $M=64, P=8$，省 8 倍。

### 3.6 气泡比例

```
理想 (无气泡): M 个 micro-batch 全并行
气泡: warmup (P-1) + cooldown (P-1) 期间部分 stage 空闲
气泡比例 = (P-1) / (M + P-1)
```

$M \gg P$ 时气泡小（如 $M=64, P=8$，气泡 $\approx 7/71 \approx 10\%$）。$M$ 小时气泡大。

### 3.7 send/recv 通信

forward 时 stage $i$ 把激活 `send` 给 stage $i+1$，`recv` 上游的输入。backward 反向 `send`/`recv` grad。async 与计算 overlap（send 不阻塞下一计算）。是 PP 的点对点通信。详见 [[Pipeline Parallel]]。

### 3.8 与 gradient checkpointing 的配合

PP 的 1F1B 仍存 $P$ 份激活/layer。若仍大，配 [[gradient checkpointing]]：stage 内只存 checkpoint 点，backward 重算。激活 $\propto \sqrt{L_{\text{stage}}}$（stage 内层重算）。是 PP 的显存栈。

### 3.9 实现的 schedules.py

```python
# megatron/core/pipeline_parallel/schedules.py
def forward_backward_pipelining_without_interleaving(...):
    num_warmup = pipeline_parallel_size - 1 - stage_id  # warmup 数
    forward_step(...)  # warmup forwards
    for i in steady:  # 1F1B
        forward_step(); backward_step()  # 交错
    for i in cooldown:  # 剩余 backward
        backward_step()
```

### 3.10 与 interleaved 的对比

1F1B：每 stage 持 1 个连续 layer 段，气泡 $\frac{P-1}{M+P-1}$。[[Interleaved schedule]]：每 stage 持 $V$ 个**分散**的 virtual stage（chunk），气泡 $\frac{P-1}{M+P-1} \cdot \frac{1}{V}$（减 $V$ 倍）。interleaved 是 1F1B 的演进（减气泡，增通信复杂度）。


## 4. 数学原理 / 公式

### 4.1 激活显存

GPipe：stage $i$ 存 $M - i$ 份激活 $\approx M \cdot A$（$A$ = 单 micro-batch 激活）。1F1B：存 $P$ 份 $\approx P \cdot A$。省 $M/P$ 倍。

### 4.2 气泡比例

$$\text{bubble} = \frac{P-1}{M+P-1}$$

$P$ stage，$M$ micro-batch。$M \gg P$ 时 $\approx (P-1)/M$（小）。$M \approx P$ 时气泡大。

### 4.3 计算量

每 micro-batch 每 stage 算 $1/P$ 的 model FLOP（PP 切层）。$M$ 个 micro-batch 总 $M \cdot \text{model FLOP}$。无重叠（每 stage 算自己 micro-batch）。计算时间 $\propto M \cdot \text{FLOP}_{\text{stage}} / \text{throughput}$。

### 4.4 通信量

每 micro-batch 每 stage 一次 `send`/`recv` 激活 $\approx B \cdot S \cdot d / P_{\text{stage}}$（激活 size）。$M$ 个 micro-batch、$P$ stage。总 P2P $\propto M \cdot B \cdot S \cdot d$（跨 stage）。详见 [[Pipeline Parallel]]。

```python
from megatron.core.pipeline_parallel import schedules
# Megatron 自动用 1F1B (无 interleaving)
loss = schedules.forward_backward_pipelining_without_interleaving(
    model, batch, forward_step, backward_step,
    pipeline_parallel_size=8, micro_batch_size=...)
# 内部: warmup -> steady 1F1B -> cooldown
```

### 5.2 三阶段示意

```python
# stage 0, P=4, M=8
warmup = P - 1 - 0  # = 3 forwards
steady = M - warmup  # = 5 (1F1B 交替)
cooldown = warmup  # = 3 backwards
for _ in range(warmup): forward_step()  # F0 F1 F2
for _ in range(steady): forward_step(); backward_step()  # F3-B0 F4-B1 ...
for _ in range(cooldown): backward_step()  # B5 B6 B7
```


## 6. 与其他知识点的关系

- **上游（依赖）**: [[Pipeline Parallel]]（PP 概念）、[[Megatron Core目录与执行链路]]（时序）、[[forward与backward]]、[[micro-batching]]、[[overlap strategy]]、[[activation memory]]。
- **下游（应用）**: [[Interleaved schedule]]（演进）、[[Megatron-LM]] 的 PP 实现、[[3D parallelism]]（与 TP/DP 组合）、[[gradient checkpointing]]（互补省显存）、[[Pipeline Parallel]] 的 GPipe 对比。
- **对比 / 易混**:
  - **1F1B vs GPipe**：GPipe 全 forward 再全 backward（激活 $\propto M$）；1F1B 交错（激活 $\propto P$）。气泡同 $\frac{P-1}{M+P-1}$。1F1B 省显存，GPipe 简单。
  - **1F1B vs [[Interleaved schedule]]**：1F1B 每 stage 1 连续段，气泡 $\frac{P-1}{M+P-1}$；interleaved 每 stage $V$ 个分散 virtual stage，气泡 $\frac{P-1}{M+P-1}/V$。interleaved 减气泡但增通信复杂度。
  - **1F1B vs [[gradient checkpointing]]**：1F1B 省 stage 间激活（交错早 backward）；gradient checkpointing 省 stage 内激活（重算）。两者正交，可叠加。


## 7. 常见误区与易错点

> [!warning] 误区 1：1F1B 减气泡
> 1F1B 不减气泡（与 GPipe 同 $\frac{P-1}{M+P-1}$）。1F1B 省**激活显存**（$M \to P$）。减气泡靠 [[Interleaved schedule]]（virtual stage）。误以为减气泡会错判。

> [!warning] 误区 2：warmup 数等于 P
> warmup 数 = $P-1-i$（stage $i$，stage 0 做 $P-1$，stage $P-1$ 做 0）。非每 stage 都 $P$。每 stage warmup 数不同（填满流水线）。

> [!warning] 误区 3：steady 1F1B 是 1 forward 紧跟 1 backward
> steady 是交错（1F-1B-1F-1B-...），但 forward 与 backward 是**不同 micro-batch**（forward 算新的，backward 算最早的缓存的）。不是同 micro-batch 的 F 后 B。是流水线交错。

> [!warning] 误区 4：1F1B 不存激活
> 1F1B 仍存 $\approx P$ 份激活（warmup 期间的）。只是从 $M$ 降到 $P$。不是零激活。仍可能大，配 gradient checkpointing 进一步减。

> [!warning] 误区 5：send/recv 阻塞
> send/recv 用 async（`isend`/`irecv`），与计算 overlap。不阻塞下一 forward。误以为阻塞会错估性能。

> [!warning] 误区 6：1F1B 与 SP 冲突
> 1F1B（PP）与 [[Sequence Parallel]]（TP 的）正交（切不同维），可共存。3D 并行（TP+PP+DP）+ SP 都开。误以为冲突会漏配。

> [!warning] 误区 7：气泡 $P-1$ 是 stage 空闲时间
> 气泡 $\frac{P-1}{M+P-1}$ 中 $P-1$ 是 warmup+cooldown 的"流水线填充/排空"时间（部分 stage 空闲）。非每 stage 空闲 $P-1$。是总气泡占比。


## 8. 延伸细节

### 8.1 1F1B 的变体

Megatron 有 1F1B with interleaving（即 [[Interleaved schedule]]）、1F1B with FP8（低精度）、1F1B with SP（sequence parallel）。变体优化不同维度（气泡/精度/显存）。

### 8.2 与 [[overlap strategy]]

1F1B 的 send/recv async 与 forward/backward overlap。是 PP 的 overlap 实战。通信与计算藏。详见 [[overlap strategy]]。

### 8.3 气泡的极限

气泡 $\frac{P-1}{M+P-1}$，$M \to \infty$ 时气泡 $\to 0$。但 $M$ 大需大 batch（显存/算力）。实际 $M$ 有限（如 64-256）。气泡 10-5%。interleaved 进一步减。
