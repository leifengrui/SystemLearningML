# 训推不一致

> **所属章节**: [[Rollout系统]]
> **所属模块**: [[06-训推系统]]
> **别名**: Training-Inference Mismatch / TIM / train-inference parity / 训推 parity / 训推发散
> **难度**: 中高级（需懂 PPO 重要性采样 + 混合精度数值 + MoE 路由 + 训推双引擎架构）
> **关联**: 本笔记聚焦"训推不一致"本身（何时/如何衡量/影响），[[forward与backward]] 讲过 Pearson/logp 一致性的基础，[[R3 rollout routing replay]] 是 MoE 路由不一致的专门解法。

## 1. 一句话定义

**训推不一致（Training-Inference Mismatch, TIM）** 指 RL/RLHF 训练中，**同一模型权重、同一输入 token**，在**训练引擎**（Megatron/DeepSpeed）算出的 logits/logprob 与**推理引擎**（vLLM/SGLang/TensorRT-LLM）算出的 logits/logprob **数值不相等**的现象。它让 PPO 重要性采样比 $\rho=\pi_{\theta_{\text{new}}}/\pi_{\theta_{\text{old}}}$ 在"权重其实没变"时也偏离 1，引入系统性偏差与方差，轻则训练效率下降，重则梯度有偏、训练发散/崩溃（尤其 MoE 模型）。

## 2. 为什么需要它（动机与背景）

RLHF 的 PPO/GRPO 训练天然有**两套计算引擎**：

- **rollout 侧**（推理引擎，vLLM/SGLang）：用当前 policy $\theta$ 生成 response，记下每个 token 的 `logprob_old` $=\log\pi_\theta(a_t\mid s_t)$、reward、value。推理引擎为吞吐做了大量优化：kernel 融合、KV cache、PagedAttention、连续批处理、bf16/fp8、自定义 RoPE/RMSNorm kernel。
- **learner 侧**（训练引擎，Megatron/DeepSpeed）：对**同一条轨迹**重算 `logprob_new`，用于算 $\rho$、advantage、PPO loss。训练引擎为反向传播优化：保留中间激活、bf16/fp8 前向 + fp32 master、张量/流水线并行、gradient checkpointing。

两套引擎**实现不同**，导致即便 $\theta$ 一模一样、输入一模一样，算出的 `logprob` 也不完全相等。这个 gap 就是 TIM。在 PPO 里，`logprob_old`（rollout 算）和 `logprob_new`（learner 算）本应来自**同一分布**，$\rho$ 才能正确反映"权重更新带来的策略变化"；TIM 让 $\rho$ 多了一个"引擎实现差异"的噪声项，污染梯度信号。

> [!tip] 为什么 SFT 不怕 TIM，RL 怕
> SFT 只用训练引擎算 forward/backward，没有"推理引擎算的 logp"参与梯度，自然无所谓训推一致。RL 才有"rollout（推理）算 logp_old + learner（训练）算 logp_new"的双引擎耦合，$\rho$ 把两套 logp 绑在一起，TIM 直接进梯度。所以**TIM 是 RL/RLHF 特有的一类问题**，SFT/预训练不涉及。

## 3. 核心概念详解

### 3.1 什么时候会有训推不一致（TIM 的来源）

TIM 不是单一原因，是**数值实现差异的叠加**。按层次分：

| 层次              | 来源                    | 例子                                                                    | 严重度             |
| --------------- | --------------------- | --------------------------------------------------------------------- | --------------- |
| **精度路径**        | 训推用不同精度/舍入策略          | 训练 bf16 前向+fp32 master；推理 bf16/fp8/int8；fp8 的不同量化方案（E4M3/E5M2、per-tensor/per-block/delayed scaling 等组合训推不等价，详见 [[FP8量化方案]]） | 中-高             |
| **算子实现**        | 同算子两套 kernel 算法不同     | RoPE：旋转矩阵构造顺序；RMSNorm：epsilon 值、平方根精度；softmax：减最大值与否、累加顺序             | 中               |
| **kernel 融合**   | 推理为速度融合多个算子，改累加顺序     | fused QKV+RoPE+softmax+fused MLP；融合后中间值精度/顺序与训练逐算子不同                  | 中               |
| **归约顺序**        | 并行/张量并行切分不同 → 归约顺序非确定 | softmax over vocab 分多卡 reduce；all-reduce 浮点非结合性 → 同输入多次前向结果不同         | 中（dense）/高（MoE） |
| **KV cache 量化** | 推理量化 KV cache 省显存     | KV int8/fp8 量化，训练无此操作                                                 | 高               |
| **位置编码 base**   | RoPE base 值或计算方式不同    | 训练 `base=10000`，推理某些实现 `base=500000` 或用 NTK-aware 缩放                  | 高（一错全错）         |
| **MoE 路由**      | router TopK 在训推选不同专家  | 数值小扰动经 TopK 离散选择放大成"选不同专家"                                            | **极高（MoE 专属）**  |

> [!note] 两类 TIM
> 1. **数值型 TIM**（dense 也有）：logits 有小数值差，但专家/token 概率分布形状大体一致，Pearson 高、绝对差小。多数算子实现差异属此类，影响相对可控。
> 2. **结构型 TIM**（MoE 特有）：router 选了**不同的专家子网络**，该 token 的激活路径完全不同，logits 差距被放大。论文实测 MoE 的训推 KL 是 dense 的 ~2.4 倍，约 10% 的 router 训推选不同专家。这是 MoE RL 训练崩溃的主因，[[R3 rollout routing replay]] 专门解决它。

### 3.2 怎么衡量训推不一致（指标体系）

TIM 不是单一标量，要从多个角度量。常用指标：

| 指标                        | 公式/定义                                                                                                  | 看什么                     | 阈值经验                                |
| ------------------------- | ------------------------------------------------------------------------------------------------------ | ----------------------- | ----------------------------------- |
| **逐 token logp 绝对差**      | $\Delta_i = \lvert \log\pi_{\text{train}}(a_i) - \log\pi_{\text{infer}}(a_i) \rvert$                   | 单点最大/平均偏差               | `max_abs_diff < 1e-3`，`mean < 1e-4` |
| **Pearson 相关系数**          | $\rho_P = \frac{\text{cov}(a,b)}{\sigma_a \sigma_b}$，$a,b$ 为训/推 logp 序列                                | 两条序列是否"共线"（形状一致）        | `> 0.99`                            |
| **KL 散度**                 | $\text{KL} = \mathbb{E}_x[\text{KL}(p_{\text{train}}\cdot\|p_{\text{infer}})]$，逐 token 算 logits 分布的 KL | 整体分布偏移                  | 接近 dense baseline                   |
| **极端 token 比例 $F(\tau)$** | $F(\tau) = \Pr\left[\frac{p_{\text{train}}(a)}{p_{\text{infer}}(a)} > \tau \text{ 或 } < 1/\tau\right]$ | 概率比超 $\tau$ 倍的 token 占比 | $F(2) < 10^{-4}$（R3 后）；无 R3 可达 0.1+ |
| **JS 散度**                 | $\text{JS} = \frac12\text{KL}(p\|m) + \frac12\text{KL}(q\|m)$，$m=(p+q)/2$                              | 对称、有界的分布差异              | 越接近 0 越好                            |
| **token_mult_prob_error** | 多 token 概率乘积的误差                                                                                        | verl/NeMo-RL 训练监控字段     | 越低越好                                |
| **ratio 偏移**              | $\rho = \exp(\log p_{\text{new}} - \log p_{\text{old}})$ 在 $\theta$ 未变时的值                              | 应 $\approx 1$，偏离即 TIM   | mean≈1，std 小                        |

#### Pearson 详解（用户重点问的）

逐 token 取训练 logp 序列 $a=(a_1,\dots,a_T)$ 与推理 logp 序列 $b=(b_1,\dots,b_T)$：

$$
\rho_P = \frac{\sum_i (a_i - \bar a)(b_i - \bar b)}{\sqrt{\sum_i(a_i-\bar a)^2}\sqrt{\sum_i(b_i-\bar b)^2}} \in [-1, 1]
$$

- **为什么用 Pearson**：衡量"是否共线"，对整体 scale 和常数偏移不敏感。RLHF 里 advantage 只看**相对序**（哪个 token 概率高/低），Pearson 贴合需求——即便推理整体偏移一个常数，只要形状对，PPO 梯度方向仍对。
- **局限**：Pearson 高但绝对差大也有问题（如推理整体平移）。所以**实务同时看两个**：`Pearson > 0.99` **且** `max_abs_diff < 1e-3`（或 `mean_abs_diff < 1e-4`）。前者保形状，后者保绝对。
- **为什么比 logp 不比 logits/loss**：logp $=\log\text{softmax(logits)}[\text{token}]$ 单 token 一个标量、逐点可比，且是 PPO 直接消费的 $\log\pi(a\mid s)$；log_softmax 减最大值数值稳定，bf16 误差小。logits 整个 vocab 向量比太严、loss 混了 target 太间接。

#### KL 与 $F(\tau)$ 详解

$$
\text{KL}_{\text{TIM}} = \mathbb{E}_{x,a\sim\text{rollout}}\left[\text{KL}\big(p_{\text{train}}(\cdot\mid x)\,\|\,p_{\text{infer}}(\cdot\mid x)\big)\right]
$$

逐 token 对整个 vocab 的概率分布算 KL，再对轨迹平均。论文用它量化 MoE vs dense 的 TIM 严重度（Qwen3-30B MoE: 1.5e-3 vs Qwen3-8B dense: 6.4e-4）。

$F(\tau)$ 关注**尾部极端不一致**——多数 token 可能差不多，但少数 token 概率差超 $\tau$ 倍，这些是破坏 PPO 的"毒点"。R3 把 $F(2)$ 从 $>0.1$ 压到 $<10^{-4}$。

> [!tip] 排查 TIM 的工程方法
> 逐层算 logp diff，二分定位到具体算子/层。常见坑：RMSNorm epsilon、RoPE base、softmax 减最大值、KV cache 量化。verl 有 `rollout_probs_diff` / `pearson` / `max_abs_diff` 等监控字段（见 [[RLHF训练监控指标详解]]）。

### 3.3 训推不一致会有什么样的影响

TIM 的影响从轻到重分几级：

**1. 梯度信号噪声化（轻度）**
$\rho = \exp(\log p_{\text{new}} - \log p_{\text{old}})$ 在 $\theta$ 未变时应 $\approx 1$。TIM 让 $\rho$ 偏离 1、引入方差。PPO clip 频率升高（更多 token 被裁断），有效梯度信号减弱，收敛变慢、样本效率降。

**2. 重要性采样有偏（中度）**
若 TIM 不是零均值噪声而是**系统性偏移**（如推理整体 logp 偏低），$\rho$ 的期望 $\neq 1$，梯度估计有偏。策略更新方向系统性跑偏，模型往错误方向优化。

**3. KL 失控 / 训练发散（重度）**
$\rho$ 方差大 → 策略更新幅度大 → $\pi_{\theta_{\text{new}}}$ 与 $\pi_{\theta_{\text{old}}}$ 的 KL 快速攀升 → target_kl 早停频繁触发 → 训练走走停停；或 KL 爆炸（[[KL explosion]]）直接发散。

**4. 训练崩溃（极重，MoE 专属）**
MoE 的结构型 TIM（router 选不同专家）让 10%+ 的 token 训推激活路径完全不同，$\rho$ 在这些 token 上剧烈偏离，梯度噪声爆炸，policy 快速退化到不可恢复（[[policy collapse]]）。论文观测：SFT+GRPO 单 mini-step，60 步后 $F(2)>0.1$，随后崩。这正是 [[R3 rollout routing replay]] 要解决的。

**5. 算法层补丁的次生问题**
为压 TIM 用 rollout correction（clip/丢弃超阈样本），但当 TIM 涨得比丢弃快时，**有效样本被丢光、训练停摆**——治标反被治标伤。

> [!warning] TIM 与 off-policy staleness 的区别（易混）
> - **TIM（训推不一致）**：$\theta$ **相同**，两套引擎算出不同 logp。根因=实现差异。R3/精度对齐解决。
> - **staleness（off-policy 偏移）**：rollout 用 $\theta_{\text{old}}$、learner 用 $\theta_{\text{new}}$，$\theta$ **不同**。根因=异步权重滞后。重要性采样 $\rho$ 修正。
> 两者都表现为 $\rho\neq 1$，但根因和解法不同。TIM 让 $\rho$ 在"不该偏的时候偏"，staleness 让 $\rho$ "该偏且要修正"。R3 只治 TIM，不替代 staleness 的 $\rho$ 修正。

## 4. 数学原理 / 公式

### 4.1 PPO 目标与 $\rho$ 的角色

$$
\mathcal{L}^{\text{PPO}} = \mathbb{E}\big[\min(\rho A,\, \text{clip}(\rho,1{-}\epsilon,1{+}\epsilon)A)\big],\quad \rho = \frac{\pi_{\theta_{\text{new}}}(a\mid s)}{\pi_{\theta_{\text{old}}}(a\mid s)} = \exp(\log p_{\text{new}} - \log p_{\text{old}})
$$

理想情况（无 TIM、无 staleness）：$\theta_{\text{new}}=\theta_{\text{old}} \Rightarrow \rho=1$。

### 4.2 TIM 引入的 $\rho$ 噪声

设 $\log p_{\text{new}} = \log p_{\text{old}} + \delta_{\text{TIM}}$（$\theta$ 未变，$\delta_{\text{TIM}}$ 纯引擎差异），则：

$$
\rho = e^{\delta_{\text{TIM}}}
$$

- 若 $\delta_{\text{TIM}}\sim\mathcal{N}(0,\sigma^2)$（零均值噪声）：$\mathbb{E}[\rho]=e^{\sigma^2/2}>1$，$\text{Var}(\rho)$ 随 $\sigma$ 指数涨 → 梯度方差爆炸。
- 若 $\delta_{\text{TIM}}$ 有非零均值 $\mu$：$\mathbb{E}[\rho]=e^{\mu+\sigma^2/2}$，梯度**有偏**。

### 4.3 KL 与 $F(\tau)$

$$
\text{KL}_{\text{TIM}} = \mathbb{E}_{x,a}\left[\sum_v p_{\text{train}}(v\mid x)\log\frac{p_{\text{train}}(v\mid x)}{p_{\text{infer}}(v\mid x)}\right]
$$

$$
F(\tau) = \Pr_{x,a\sim\text{rollout}}\left[\frac{p_{\text{train}}(a\mid x)}{p_{\text{infer}}(a\mid x)} > \tau \;\text{或}\; < \frac1\tau\right]
$$

$F(\tau)$ 抓尾部极端，KL 抓整体均值，Pearson 抓形状，max_abs_diff 抓绝对，四者互补。

### 4.4 MoE 的 TIM 放大

dense：$\log p$ 差 $\approx$ 数值噪声 $\mathcal{O}(\epsilon_{\text{bf16}})$，Pearson $>0.999$。

MoE：router mask $I_{\text{train}}\neq I_{\text{infer}}$ 时，激活专家子网络不同，输出 $y$ 差距 $\mathcal{O}(1)$（不是 $\mathcal{O}(\epsilon)$），logits 差被放大。约 10% 的 router 不一致 → MoE KL 是 dense 的 ~2.4 倍。

## 5. 代码示例

```python
import numpy as np

def measure_tim(logp_train, logp_infer):
    """logp_train/logp_infer: 同一条轨迹、逐 token 的 logp 数组 (T,)
    返回 TIM 的多维度指标。"""
    a, b = np.asarray(logp_train), np.asarray(logp_infer)
    diff = a - b
    # 1. 绝对差
    max_abs_diff = np.max(np.abs(diff))
    mean_abs_diff = np.mean(np.abs(diff))
    # 2. Pearson 相关系数
    pearson = np.corrcoef(a, b)[0, 1]
    # 3. ratio 偏移 (θ 未变时 rho 应 ≈1)
    rho = np.exp(diff)
    rho_mean, rho_std = rho.mean(), rho.std()
    # 4. 极端 token 比例 F(τ): 概率比超 τ 倍的占比
    def F(tau):
        return np.mean((rho > tau) | (rho < 1.0 / tau))
    # 5. KL (逐 token 的 logits 分布 KL，这里简化为单点 logp 差的近似)
    kl_approx = np.mean(diff)  # 严格 KL 需整个 vocab 分布，这里仅示意
    return dict(
        max_abs_diff=max_abs_diff,
        mean_abs_diff=mean_abs_diff,
        pearson=pearson,
        rho_mean=rho_mean, rho_std=rho_std,
        F2=F(2.0),
        kl_approx=kl_approx,
    )

# 模拟：θ 未变，纯引擎数值差异
np.random.seed(0)
T = 1000
logp_true = np.random.randn(T) * 2 - 5          # 真实 logp
logp_train = logp_true + np.random.randn(T)*1e-3  # 训练引擎: 小噪声
logp_infer = logp_true + np.random.randn(T)*1e-3  # 推理引擎: 小噪声 (dense 良好情况)
print("dense 良好情况:", {k: round(v,4) for k,v in measure_tim(logp_train, logp_infer).items()})

# 模拟 MoE 结构型 TIM: 10% 的 token 训推选不同专家 → 大偏差
moE_diff = np.random.randn(T)*1e-3
moE_diff[np.random.choice(T, 100, replace=False)] += np.random.randn(100)*1.5  # 10% 大偏差
logp_train_moe = logp_true
logp_infer_moe = logp_true - moE_diff
print("MoE 结构型 TIM:", {k: round(v,4) for k,v in measure_tim(logp_train_moe, logp_infer_moe).items()})
```

预期：dense 情况 Pearson≈1、max_abs_diff 小、F2≈0；MoE 情况 F2≈0.1、KL 大、rho_std 大。

- **对比 / 易混**:
  - **TIM vs staleness**：见 §3.3 易混警示。TIM=同 $\theta$ 引擎差异，staleness=不同 $\theta$ 权重滞后。
  - **数值型 TIM vs 结构型 TIM**：前者 logits 小扰动（dense 也有），后者 router 选不同专家（MoE 专属），解法不同（精度对齐 vs R3）。
  - **TIM 的 $\rho$ vs staleness 的 $\rho$**：都表现为 $\rho\neq1$，但前者是假性偏移（应消除），后者是真实偏移（应修正）。
  - **TIM vs bitwise determinism**：bitwise 一致是最强约束（逐 bit 相同），TIM 容忍小数值差只要不破坏 PPO；R3 是"够用且轻"的折中。

## 7. 常见误区与易错点

> [!warning] 误区 1：以为 SFT 也要管训推一致
> SFT 只用训练引擎，没有"推理引擎算的 logp"进梯度，无所谓 TIM。TIM 是 RL/RLHF 特有（双引擎耦合 $\rho$）。

> [!warning] 误区 2：只看 Pearson 不看绝对差
> Pearson 高但绝对差大也有问题（整体平移）。务必同时看 `Pearson>0.99` 且 `max_abs_diff<1e-3`。

> [!warning] 误区 3：把 TIM 当 staleness 治
> TIM 是同 $\theta$ 引擎差异，用精度对齐/R3 治；staleness 是不同 $\theta$ 权重滞后，用 $\rho$ 修正/weight sync 治。混了就开错药。

> [!warning] 误区 4：以为 dense 模型也有结构型 TIM
> 结构型 TIM（选不同专家）是 MoE 专属。dense 只有数值型 TIM，相对可控。

> [!warning] 误区 5：靠 rollout correction 丢样本治 TIM
> 治标不治本，TIM 涨快时样本丢光训练停摆。应从根因（精度对齐/R3）消除，而非算法层裁断。

> [!warning] 误区 6：忽略 KV cache 量化引入的 TIM
> 推理量化 KV cache（int8/fp8）省显存，但训练无此操作，是高频 TIM 来源。排查时要查推理的 KV 量化配置。

> [!warning] 误区 7：RoPE base / RMSNorm epsilon 不对齐
> 这两个是"一错全错"的高频坑：RoPE base 不同 → 所有位置编码错；RMSNorm epsilon 不同 → 每层归一化都偏。部署时务必对齐训练超参。

### 8.1 各框架的 TIM 监控字段

- **verl**：`actor/rollout_probs_diff_mean`、`actor/rollout_probs_pearson`、`actor/rollout_probs_max_abs_diff`，每步打印。

- **OpenRLHF**：`actor/kl_loss`、`actor/clipfrac` 间接反映 TIM。

### 8.2 消除 TIM 的手段分层

| 层次 | 手段 | 适用 |
|---|---|---|
| 精度对齐 | 训推同精度（bf16+fp32 master）、统一 fp8 方案 | dense 数值型 TIM |
| 算子对齐 | 训推同 kernel/同归约顺序、关非确定算法 | dense 数值型 TIM |
| 超参对齐 | RoPE base、RMSNorm eps、softmax 减最大值统一 | 全局配置型 TIM |
| bitwise determinism | 同实现、定 determinism、禁并行非确定 | 最严，性能开销大 |
| **R3** | 推理记录路由→训练回放 | **MoE 结构型 TIM** |
| KV 量化对齐 | 推理关 KV 量化，或训练也模拟量化 | KV cache 引入的 TIM |

### 8.3 TIM 与训推一致性工程的关系

[[训推分离]] 里的 weight sync 保证"训推**权重**一致"，TIM 讲的是"权重一致后**前向计算**还可能不一致"。两者正交：weight sync 是参数层，TIM 是计算层。完整的一致性工程要两层都做（权重同步 + 前向对齐/R3）。

### 8.4 趋势

- MoE 成 RLHF 主力（DeepSeek-V3/R1、Qwen3、Kimi K2），结构型 TIM 普遍化，R3 从可选变必备。
- 推理引擎原生支持路由导出（SGLang 已支持，vLLM 跟进），TIM 治理门槛降低。
- 训推一致性成为 RL 系统的基准验收项（pearson/max_abs_diff 进 CI）。
