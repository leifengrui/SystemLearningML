# 损失函数分类

> **所属章节**: [[训练工程基础]]
> **所属模块**: [[01-ML深度学习基础]]
> **难度**: 基础~中级

## 1. 一句话定义

**损失函数（loss function，损失函数）** 是衡量预测 $\hat y$ 与真实 $y$ 差距的**可微标量**，训练就是最小化它；按任务分为**回归**、**分类**、**排序**、**生成/概率**、**对比学习**、**RL** 六大类，每类有各自的数学形式与适用场景。
> [!note] 解答：为什么叫"损失函数"？
> 
> **白话**：你预测错了就要"赔"——"损失"就是衡量**这次预测错得有多惨**的那个数，越小越好。"函数"是因为它把 (预测 $\hat y$, 真实 $y$) 这一对输入映射成一个标量。合起来：**损失函数 = 把"预测错误代价"翻译成一个可微标量的函数**，训练就是想办法让它最小。
> 
> **词源**：loss function 不是深度学习发明的词，来自**统计决策论**（Wald, 1950）与贝叶斯决策。那里把每个决策的"代价"记成 loss，期望代价叫 risk（风险）。机器学习沿用此术语：单样本代价 = loss，全样本平均 = 经验风险（empirical risk），训练 = 最小化经验风险。中文"损失函数"是 loss function 的直译，港台也译"损失函式"。
> 
> **容易混的四个近义词**（务必分清）：
> 
> | 术语 | 定义 | 例子/备注 |
> |---|---|---|
> | loss function（损失函数） | **单样本**的代价 $\ell(y,\hat y)$ | CE、MSE 都是单样本公式 |
> | cost function（代价函数） | 数据集上的**汇总**，常为均值 $J=\frac1N\sum_i\ell_i$ | 很多人把 loss/cost 混用，严谨上 cost 是聚合后的 |
> | objective function（目标函数） | 最泛，任何优化的目标，可含正则/罚项 | $\min_\theta\ J+\lambda\|\theta\|_2^2$ 整个就是 objective；不一定"越小越好"，也可能是 maximize |
> | risk（风险/期望损失） | loss 在**真实分布**上的期望 $R(\theta)=\mathbb{E}_{(x,y)}[\ell]$ | 真分布未知，故用经验风险 $\frac1N\sum\ell$ 近似——这就是 ERM（经验风险最小化）原理 |
> 
> **三个基本性质**（为什么 loss 这么定义合理）：
> 
> 1. **非负**：$\ell(y,\hat y)\ge 0$，错得越离谱值越大。
> 2. **零损失**：$\ell(y,y)=0$，预测完全对则零代价（CE/MSE 都满足）。
> 3. **可微**：能求梯度，优化器才用得上（accuracy 不可微所以只能当 metric，不能当 loss，见 §6 对比）。
> 
> **一个直觉比喻**：考试改卷。一题答错扣几分 = 该题的 loss；全卷总扣分 = cost（平均扣分率）；学校的排分规则 + 作文扣字等"目标" = objective。学生（模型）要做的是让总扣分最少。
> 
> **一句话收尾**：loss 直译"损失"，就是把"预测错得多惨"量化成一个能优化的标量函数；叫"损失"而不叫"误差"是强调它是**带代价含义的目标**（要最小化），不只是差的度量。详见本笔记 §2（动机）、§4（数学原理）。关联 [[SGD]]、[[Adam与AdamW]]、[[backward过程]]、[[梯度与Jacobian]]。
## 2. 为什么需要它（动机与背景）

优化器（[[SGD]]/[[Adam与AdamW]]）需要一个**可微的标量目标**才能算梯度更新参数。损失函数就是这个目标：它把"预测好不好"翻译成"一个数"，越小越好。

不同任务"好不好"的定义不同：回归看距离、分类看概率匹配、生成看似然、RL 看回报期望。所以损失函数必须**按任务选**，选错则优化方向全错。

## 3. 核心概念详解

### 3.1 六大类总表

| 类别 | 典型 loss | 任务 | 一句话 |
|---|---|---|---|
| 回归 | MSE, MAE, Huber, Smooth L1 | 数值预测 | 拉近预测值与真值 |
| 分类 | CE, BCE, Focal | 离散标签 | 让正确类概率高 |
| 排序 | pairwise, listwise | 检索/推荐 | 正样本排序高于负 |
| 生成/概率 | NLL, CE, KL | 语言模型/生成 | 最大化真序列似然 |
| 对比 | InfoNCE, Triplet | 表征学习 | 正对拉近、负对推开 |
| RL | policy loss, value loss | 强化学习 | 最大化期望回报 |

### 3.2 回归类

- **MSE（L2）**：$\frac1N\sum(\hat y-y)^2$。放大大误差（平方），对噪声敏感。
- **MAE（L1）**：$\frac1N\sum|\hat y-y|$。对异常值鲁棒，但在 0 不可导（用次梯度）。
- **Smooth L1 / Huber**：小误差用平方、大误差用线性，兼顾两者。检测框回归常用。

### 3.3 分类类

- **Cross-Entropy（CE，交叉熵）**：分类主力。多类用 `F.cross_entropy`（内含 logsoftmax+nll），二类用 `BCEWithLogits`。

$$
L_{\text{CE}}=-\sum_i y_i\log\hat p_i
$$

- **Focal Loss**：$\text{CE}\times(1-\hat p)^\gamma$，降权易分样本，解决类别极不平衡（检测用）。
- **Label Smoothing**：把 one-hot 的 1 改成 $1-\epsilon$、其余加 $\epsilon/K$，防过自信、提泛化。LLM 预训练标配（$\epsilon=0.1$）。

### 3.4 生成/概率类（LLM 核心）

语言模型训练 = **最大化真 token 的对数似然**，等价于最小化 NLL/CE：

$$
L=-\frac1N\sum_t \log p_\theta(x_t\mid x_{<t})
$$

- 即每个位置预测下一 token，对真 token 取负对数概率求平均。
- [[perplexity]]（困惑度）= $\exp(L)$，是 NLL 的指数化、更直观的语言模型指标。
- 与 KD/蒸馏的 KL：让 student 分布逼近 teacher（[[DPO]] 也用 KL）。

### 3.5 对比类

- **InfoNCE**：把"正样本对"拉近、"负样本对"推开，是 SimCLR/CLIP/对比预训练的主力。
- **Triplet**：anchor-positive-negative，让正距离 < 负距离 − margin。
- 常配温度 $\tau$ 缩放相似度。

### 3.6 RL 类

- **policy loss**（[[PPO]]）：$-\mathbb{E}[\min(r_t A_t,\text{clip}(r_t)A_t)]$，最大化带 clip 的优势。
- **value loss**：拟合 $V(s)$ 的 MSE。
- **entropy bonus**：$-\mathbb{E}[H(\pi)]$，鼓励探索防崩塌。

## 4. 数学原理 / 公式

### 交叉熵与 KL 的关系

真实分布 $p$、预测 $\hat p$：

$$
H(p,\hat p)=H(p)+D_{\text{KL}}(p\|\hat p)
$$

标签 one-hot 时 $H(p)=0$，故 $\text{CE}=D_{\text{KL}}$ → 最小化 CE = 让预测逼近真分布。这就是分类用 CE 的数学根。

### CE + softmax 的数值稳定

直接算 $\log\text{softmax}$ 会因 exp 溢出。稳定公式（log-sum-exp trick）：

$$
\log\hat p_i = z_i - \log\sum_j e^{z_j},\quad \text{减去 } \max(z)
$$

`F.cross_entropy` 内部已做，**不要手写 softmax 再 log**，否则数值不稳。

### MSE 的概率解释

假设噪声 $y=f(x)+\epsilon,\ \epsilon\sim\mathcal N(0,\sigma^2)$，则最大似然 → 最小化 MSE。所以 MSE = 高斯噪声假设下的 MLE。

### NLL 与 perplexity

$$
\text{PPL}=\exp\!\left(-\frac1N\sum_t\log p(x_t|x_{<t})\right)=\exp(L_{\text{NLL}})
$$

PPL 越低，模型对真序列越"不困惑"。

## 5. 代码示例

```python
import torch
import torch.nn.functional as F

logits = torch.randn(4, 10)          # 4 样本 10 类
target = torch.tensor([1, 3, 0, 7])  # 真标签

# 1. 多分类 CE（内含 logsoftmax+nll，数值稳定）
loss = F.cross_entropy(logits, target, label_smoothing=0.1)   # LLM 常带 smoothing
print(loss.item())

# 2. 二分类 BCE（logits 输入，内部 sigmoid）
logit2 = torch.randn(4)
y2 = torch.tensor([1.0, 0.0, 1.0, 1.0])
loss_bce = F.binary_cross_entropy_with_logits(logit2, y2)

# 3. 回归
pred = torch.randn(4, 1); y_reg = torch.randn(4, 1)
mse = F.mse_loss(pred, y_reg)
smooth_l1 = F.smooth_l1_loss(pred, y_reg)        # Huber，检测框回归常用

# 4. LLM 下一 token 预测（本质就是 CE）
# logits: (B, T, vocab) ; target: (B, T) 的 token id
B, T, V = 2, 8, 1000
lm_logits = torch.randn(B, T, V)
lm_target = torch.randint(0, V, (B, T))
lm_loss = F.cross_entropy(
    lm_logits.reshape(-1, V),     # (B*T, V)
    lm_target.reshape(-1))        # (B*T,)
ppl = torch.exp(lm_loss)          # 困惑度
print(f"loss={lm_loss.item():.4f} ppl={ppl.item():.2f}")

# 5. Focal（类别不平衡）
p = F.softmax(logits, -1)
pt = p[range(4), target]
focal = (-(1 - pt) ** 0.25 * pt.log()).mean()

# 6. 错误示范：手写 CE 数值不稳
# p = F.softmax(logits, -1); loss = -p[range(4),target].log().mean()   # 不要这样，易溢出
```

## 6. 与其他知识点的关系

- **上游（依赖）**: [[backward过程]]（loss 是反向的根节点）、[[梯度与Jacobian]]（loss 对参数求梯度）。
- **下游（应用）**: [[SGD]]/[[Adam与AdamW]]（最小化 loss）、各类任务训练、[[PPO]]（policy/value loss）、[[DPO]]（用偏好 loss 替代 RM）。
- **对比 / 易混**:
  - **CE vs BCE**：多类 vs 二类，logits 维度不同。
  - **CE vs KL**：one-hot 标签下等价，软标签下 CE 含 KL。
  - **MSE vs CE**：回归用 MSE（高斯噪声 MLE）、分类用 CE（多项 MLE），不可互换。
  - **loss vs metric**：loss 要可微用于优化；metric（accuracy/F1）可能不可微，只评估。

## 7. 常见误区与易错点

> [!warning] 误区清单
> 1. **手写 CE 不稳定**：先 softmax 再 log 会溢出，必须用 `F.cross_entropy`（log-sum-exp trick）。
> 2. **CE 忘 label_smoothing**：LLM 预训练默认 0.1 smoothing，不加易过自信、泛化降。
> 3. **reduction=mean vs sum**：默认 mean，sum 会让 batch 越大 loss 越大、等效 lr 变；要统一。
> 4. **分类用 MSE**：用 MSE 训分类收敛慢且差，分类必用 CE（梯度量级合理）。
> 5. **忽略类别不平衡**：极不平衡数据用裸 CE 会让模型全预测多数类，用 Focal 或加权重。
> 6. **logits 维度错**：LLM 的 CE 要 reshape 成 `(B*T, V)` 与 `(B*T,)`，否则广播错。
> 7. **loss 看着降但 metric 不升**：可能 loss 与任务目标不一致，要监控真实 metric。

## 8. 延伸细节

### 8.1 label smoothing 的代价

smoothing 让模型留一点概率给其他类，降低过自信、提泛化，但**损伤 PPL**（因为不是纯 MLE）。所以"预训练加 smoothing、评估 PPL 时要去掉"或分开报。

### 8.2 DPO 的偏好 loss

[[DPO]] 不显式训 reward model，直接用 Bradley-Terry 偏好概率导出的 loss：

$$
L=-\log\sigma\!\left(\beta\log\frac{\pi(y_w)}{\pi_{\text{ref}}(y_w)}-\beta\log\frac{\pi(y_l)}{\pi_{\text{ref}}(y_l)}\right)
$$

把 RLHF 的 RM+PPO 压成一步分类式 loss。

### 8.3 contrastive 的温度

InfoNCE 里 $\tau$ 太小梯度集中在最难的负样本、训练不稳；太大失去区分。CLIP 用 0.07（可学）。

### 8.4 loss scaling 与 fp16

[[loss scaling]] 把 loss 乘大数防 fp16 梯度下溢 —— 因为是对 loss 整体操作，故 loss 形式不影响 scaling 可行性。
