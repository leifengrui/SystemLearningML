# 过拟合与欠拟合

> **所属章节**: [[训练工程基础]]
> **所属模块**: [[01-ML深度学习基础]]
> **难度**: 基础

## 1. 一句话定义

**过拟合（overfitting，过拟合）** 是模型把训练数据里的**噪声/细节也学了**，导致训练 loss 低但验证/测试差（泛化差）；**欠拟合（underfitting，欠拟合）** 是模型**连训练数据的基本规律都没学到**，训练 loss 都降不下去；两者都是"模型容量 vs 数据/正则"失衡的表现。

## 2. 为什么需要它（动机与背景）

训练的目标不是"在训练集上 loss 低"，而是"在新数据上表现好"（泛化）。但模型有"记忆力"：

- 容量太大 + 数据少 → 记住每个样本的噪声 → 过拟合（训练好、测试崩）。
- 容量太小 + 数据复杂 → 学不动规律 → 欠拟合（训练都差）。

诊断这两个状态、对症调节，是训练调参的核心技能。LLM 时代因数据海量、模型大，过拟合形态变化（见 8.3），但原理不变。

## 3. 核心概念详解

### 3.1 判读训练/验证曲线

| 现象 | 训练 loss | 验证 loss | 诊断 |
|---|---|---|---|
| 欠拟合 | 高，降不下去 | 高 | 容量不足 / 训不够 / lr 太小 |
| 正常 | 低 | 低，跟随训练 | 好 |
| 过拟合 | 低 | 训练低但验证回升 | 容量过大 / 数据少 / 正则不够 |

**过拟合的标志**：验证 loss 在某点开始**回升**而训练 loss 继续降 → 这点就是该 early stop 的位置。

### 3.2 模型容量（capacity）

容量 = 模型能表达的函数复杂度。参数多、层数深、非线性强 → 容量大。容量与数据复杂度匹配才好：容量 << 数据 → 欠拟合；容量 >> 数据 → 过拟合。

### 3.3 正则化手段

抑制过拟合的常用手段：

| 手段 | 机制 | 见 |
|---|---|---|
| [[SGD]]/[[Adam与AdamW]] 的 weight decay | 拉参数往 0 | AdamW weight_decay=0.1 |
| Dropout | 随机失活神经元 | CV/早期 Transformer |
| 数据增强 | 扩充有效数据 | CV flip/crop |
| early stopping | 验证回升就停 | 通用 |
| label smoothing | 软化标签防过自信 | LLM 预训练 |
| layer norm / 残差 | 间接稳训练 | Transformer |
| 减小模型 / 加数据 | 根本平衡容量 | — |

### 3.4 偏差-方差权衡（bias-variance tradeoff）

- **偏差（bias）高** → 欠拟合（模型假设太简单）。
- **方差（variance）高** → 过拟合（对训练数据太敏感）。
- 正则增方差降、偏差升，要找平衡点。

## 4. 数学原理 / 公式

### bias-variance 分解

对回归 $y=f(x)+\epsilon$，期望误差（以某 $x$ 为例）：

$$
\mathbb{E}[(y-\hat f(x))^2] = \underbrace{(\mathbb{E}[\hat f]-f)^2}_{\text{bias}^2} + \underbrace{\mathbb{E}[(\hat f-\mathbb{E}[\hat f])^2]}_{\text{variance}} + \underbrace{\sigma^2}_{\text{noise}}
$$

- bias：模型假设与真规律的偏差 → 欠拟合来源。
- variance：不同训练集训练出的模型波动 → 过拟合来源。
- $\sigma^2$：数据噪声，不可降。

容量增 → bias 降、variance 升，U 形总误差，最低点即平衡。

### 过拟合的数学本质

参数多到能"插值"训练数据（loss→0），但学到的函数 wiggle 大，泛化差。深度网络容量极高却仍泛化好，是深度学习理论未解之谜（见 8.2 double descent）。

## 5. 代码示例

```python
import torch
import torch.nn as nn

# 模拟：小数据 + 大模型 = 过拟合
torch.manual_seed(0)
N = 50
X = torch.randn(N, 1)
Y = torch.sin(X*3) + 0.1*torch.randn(N, 1)     # 真规律+噪声
Xv, Yv = torch.randn(N,1), torch.sin(Xv*3)+0.1*torch.randn(N,1)

big = nn.Sequential(nn.Linear(1,512), nn.ReLU(), nn.Linear(512,1))  # 大模型
opt = torch.optim.AdamW(big.parameters(), lr=1e-3, weight_decay=0)  # 不正则
lossf = nn.MSELoss()
tr, va = [], []
for ep in range(2000):
    opt.zero_grad(); l = lossf(big(X), Y); l.backward(); opt.step()
    if ep % 100 == 0:
        with torch.no_grad():
            tr.append(l.item()); va.append(lossf(big(Xv), Yv).item())
# 观察：训练 loss→0，验证 loss 先降后升 = 过拟合
print("train", [f"{x:.2f}" for x in tr[:8]])
print("val  ", [f"{x:.2f}" for x in va[:8]])

# 对策：加 weight_decay / dropout / early stop
big2 = nn.Sequential(nn.Linear(1,64), nn.ReLU(), nn.Dropout(0.3),
                     nn.Linear(64,1))
opt2 = torch.optim.AdamW(big2.parameters(), lr=1e-3, weight_decay=1e-2)
```

## 6. 与其他知识点的关系

- **上游（依赖）**: [[损失函数分类]]（用 loss 衡量）、[[SGD]]/[[Adam与AdamW]]（weight decay 是正则）。
- **下游（应用）**: 正则化设计、early stopping、checkpoint 选最优、[[layer norm]]、dropout、数据增强。
- **对比 / 易混**:
  - **过拟合 vs 训练 loss 低**：训练 loss 低不一定是过拟合，要看验证是否回升。
  - **欠拟合 vs lr 太小**：欠拟合也可能是 lr 太小/训不够，不一定是容量不足，要先排查训练问题。
  - **过拟合 vs 数值不稳**：loss 验证回升也可能是 lr 太大震荡，不是过拟合，要看曲线形态。

## 7. 常见误区与易错点

> [!warning] 误区清单
> 1. **训练 loss 低就停**：要看验证，训练好不代表泛化好。
> 2. **验证 loss 一升就 panic**：可能是 lr 震荡不是过拟合，多看几步、平滑曲线再判。
> 3. **正则越大越好**：过强正则 → 欠拟合，训练都降不动。
> 4. **LLM 不过拟合所以不防**：LLM 数据大不易过拟合，但 SFT/微调小数据时仍会过拟合（见 8.3）。
> 5. **只看最终 loss 选模型**：应按验证最优 checkpoint，不是最后一步。
> 6. **dropout 在推理不关**：推理必须 `model.eval()` 关 dropout/BN，否则结果不稳。

## 8. 延伸细节

### 8.1 early stopping 的实现

监控验证 metric，连续 N 个 eval 不提升就停训练，并保存最优 checkpoint。是廉价有效的防过拟合手段。

### 8.2 double descent（双重下降）

现代深度学习发现：容量继续增大越过"插值点"后，测试误差**再次下降**（model-wise / epoch-wise double descent）。颠覆了经典 U 形 —— 大模型在数据够时反而更泛化。LLM 的"大而不过拟合"部分源于此。

### 8.3 LLM 的过拟合形态

预训练：数据海量，过拟合不明显，反靠数据质量。**SFT/微调**：数据少（几千~几万条），极易过拟合（几 epoch 就验证崩），故 SFT 常 1~3 epoch + early stop、低 lr。RLHF 同理要防 policy collapse。

### 8.4 数据质量 > 数量

脏数据让模型学错规律，比过拟合更致命。LLM 时代强调数据清洗、去重、过滤，胜过堆正则。
