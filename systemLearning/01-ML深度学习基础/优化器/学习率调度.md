# 学习率调度

> **所属章节**: [[优化器]]
> **所属模块**: [[01-ML深度学习基础]]
> **难度**: 中级

## 1. 一句话定义

**学习率调度（learning rate schedule，LR schedule，学习率调度）** 是在训练过程中按 step/epoch 动态改变学习率 $\eta_t$ 的策略；现代 LLM 标准配方是 **warmup（线性升温）+ cosine decay（余弦退火）**，前期小步爬升防发散、后期平滑下降精修，是训练稳定性和最终质量的关键旋钮。

## 2. 为什么需要它（动机与背景）

固定 lr 的问题：

1. **前期太大就发散**：初始 $m,v$ 未稳、参数随机，大 lr 直接 loss spike 甚至 NaN。
2. **后期太大就震荡**：接近极小值时大 lr 会越过最优来回弹跳，loss 不降反升。
3. **全程偏小则慢**：保守的小 lr 收敛极慢。

所以要让 lr **随训练阶段自适应**：开头小（warmup 稳住）、中期大（快速下降）、后期小（精修收敛）。这就是调度。

> [!note] lr 是最重要的超参
> 经验上 lr 比优化器选择、batch、正则都更影响结果。"调不好 lr，其它都白搭"。

## 3. 核心概念详解

### 3.1 三大主流策略

| 策略 | 形状 | 适用 | 一句话 |
|---|---|---|---|
| **warmup** | 线性升 0→peak | 通用前期 | 防 AdamW 前期发散 |
| **cosine decay** | 余弦降 peak→min | LLM 主力 | 平滑退火，后期精修 |
| **linear decay** | 线性降 peak→0 | 通用/微调 | 简单有效 |
| **step decay** | 阶梯降 | CV 经典 | 每 N epoch 砍 0.1 |
| **constant** | 不变 | RL/pretrain 某些阶段 | 简单稳定 |

### 3.2 Warmup（升温）

$$
\eta_t = \eta_{\text{peak}}\cdot\frac{t}{T_w},\quad t\le T_w
$$

- $T_w$：warmup 步数，LLM 常用 2000~1% 总步数。
- 作用：Adam 的 $v_t$ 前期从 0 起步、偏差校正后步长仍不稳；小 lr 给系统"热身"，等 $m,v$ 收敛后再放步长。

> [!tip] 为什么 LLM 必须 warmup 而 CV 不一定要
> AdamW 的 $v$ 初始为 0，前期 $\sqrt{v}$ 很小 → 步长大且噪声大。LLM lr 峰值大（3e-4）+ 模型深，不 warmup 必 spike；CV 用 SGD+小 lr(0.1) 时 $v$ 不存在，可省略。

### 3.3 Cosine Decay（余弦退火）

$$
\eta_t = \eta_{\min} + \tfrac{1}{2}(\eta_{\text{peak}}-\eta_{\min})\left(1+\cos\frac{\pi(t-T_w)}{T-T_w}\right),\quad t>T_w
$$

- $T$：总步数，$T_w$：warmup 结束。
- $\eta_{\min}$：终值，常 0 或 0.1×peak。
- 形状：先缓降、中段快降、末段再缓 → 前/后精修、中段快速接近。

LLM 预训练几乎全用 cosine，因为平滑、后期收敛质量好。

### 3.4 Linear Decay

$$
\eta_t = \eta_{\text{peak}}\cdot\frac{T-t}{T-T_w}
$$

简单线性降到 0。微调、某些预训练用，效果接近 cosine 但更易调（少一个 min 参数）。

### 3.5 WSD（Warmup-Stable-Decay）

新趋势：warmup→长 plateau（常数 peak）→末段快速 decay。可**无限续训**（plateau 段随时停），Llama3、Qwen 系用此变体。

### 3.6 Restart / 多周期

Cosine restart（SGDR）：周期性把 lr 调回 peak，逃离局部最优。LLM 少用（不稳定），CV 偶用。

## 4. 数学原理 / 公式

### cosine 的导数

$\frac{d\eta}{dt}\propto\sin(\cdot)$：中段斜率最大，首尾斜率小。等价于"开始/结束慢、中间快"，契合训练三阶段（探索→收敛→精修）的直觉。

### warmup 的偏差校正互补

AdamW 偏差校正 $\hat v=v/(1-\beta_2^t)$，前期 $1-\beta_2^t\to0$ 让 $\hat v$ 放大、步长缩小已部分防发散；但 $\beta_2=0.999$ 时 $1-\beta_2^t$ 在前几千步仍很小，仅靠它不够，warmup 是显式补强。

### lr 与 batch 的关系

线性缩放：$\eta_{\text{peak}}\propto\text{batch}$。大 batch 噪声小、等效步长可放大。LLM 用百万 token batch，peak lr 可到 3e-4~6e-4。

## 5. 代码示例

```python
import torch
from torch.optim.lr_scheduler import LambdaLR, CosineAnnealingLR
import math

model = torch.nn.Linear(10, 2)
opt = torch.optim.AdamW(model.parameters(), lr=3e-4, weight_decay=0.1)

# 标准 warmup + cosine（手写 LambdaLR，最常用）
def warmup_cosine(step, total=10000, warmup=500, min_lr=0.0):
    peak = 1.0
    if step < warmup:
        return step / warmup                    # 线性升温到 peak
    progress = (step - warmup) / (total - warmup)
    return min_lr + 0.5*(peak-min_lr)*(1+math.cos(math.pi*progress))

scheduler = LambdaLR(opt, lr_lambda=lambda s: warmup_cosine(s, 10000, 500))

# 训练循环
for step in range(20):
    opt.zero_grad(set_to_none=True)
    loss = model(torch.randn(8,10)).pow(2).sum()
    loss.backward()
    opt.step()
    scheduler.step()                            # 每个 step 后调
    print(step, opt.param_groups[0]['lr'])

# transformers 的实现（LLM 常用）

```

## 6. 与其他知识点的关系

- **上游（依赖）**: [[SGD]]、[[Adam与AdamW]]（调度作用在优化器的 lr 上）、[[Momentum]]（动量改变等效 lr）。
- **下游（应用）**: 训练稳定性（防 spike）、[[mixed precision training]]（lr 与 loss scaling 协同）、PPO/RLHF（policy lr 调度）。
- **对比 / 易混**:
  - **scheduler.step() 位置**：PyTorch 推荐 `optimizer.step()` 后调 `scheduler.step()`；放反了第一步 lr 错。
  - **per-param-group lr**：不同参数组可不同调度（如 bias/no-decay 组 lr 更大）。
  - **lr schedule vs lr finder**：finder 是**选**峰值 lr 的工具（从小到大扫找最陡下降段），schedule 是**用** lr 的策略。

## 7. 常见误区与易错点

> [!warning] 误区清单
> 1. **不 warmup 直接 peak**：AdamW 前期必 spike/NaN，LLM 训练头号坑。
> 2. **`scheduler.step()` 顺序错**：应在 `optimizer.step()` 之后；放之前第一步用了未更新 lr。
> 3. **`num_training_steps` 算错**：= epochs × steps_per_epoch，估错导致 cosine 提前归零或没降够。
> 4. **cosine 的 min_lr 设 0 太狠**：末期 lr=0 等于停止学习，续训不涨；常留 0.1×peak。
> 5. **断点续训不存 scheduler state**：scheduler 有内部 step 计数，要随 state_dict 存，否则 lr 跳回开头。
> 6. **RL 里也照搬 cosine**：on-policy RL 数据分布随 policy 变，常**用常数 lr** 或很缓的 decay，否则后期学不动新数据。
> 7. **改 batch 不改 peak lr**：违反线性缩放，大 batch 收敛慢。

## 8. 延伸细节

### 8.1 lr finder

Leslie Smith 的方法：lr 从 1e-7 指数升到 10，画 loss-lr 曲线，选"loss 下降最陡"前的 lr 作 peak。是定峰值 lr 的实用工具。

### 8.2 μP（maximal update parametrization）

通过精巧的参数化让**最优 lr 随模型宽度不变**，于是小模型调好的 lr 直接用到大模型（Llama/Qwen 用 μP 调参省成本）。这是 lr 调度的"元优化"。

### 8.3 cooldown

某些配方在 cosine 末尾加一段 lr 恒为 min 的"cooldown"，做最后精修、稳定收尾。

### 8.4 WSD 的可续训优势

WSD plateau 段 lr 恒定，**任何点都可停**并接一段 decay 收尾，适合"数据持续增加、训到哪算哪"的大模型场景，比固定 cosine 更灵活。

### 8.5 与 PPO 稳定性

PPO 的 policy lr 常用常数或线性衰减；KL 爆炸时降 lr 是应急手段。见 [[PPO]]、[[KL penalty]]。
