# 梯度裁剪

> **所属章节**: [[优化器]]
> **所属模块**: [[01-ML深度学习基础]]
> **难度**: 中级

## 1. 一句话定义

**梯度裁剪（gradient clipping，梯度裁剪）** 是在 `loss.backward()` 之后、`optimizer.step()` 之前，对梯度做**缩放或截断**以防其过大导致参数更新爆炸；分 **按范数裁剪（clip by norm）** 和 **按值裁剪（clip by value）** 两类，LLM/RNN/PPO 训练的标准稳定手段。

## 2. 为什么需要它（动机与背景）

深度模型梯度可能突然暴涨：

- **梯度爆炸（gradient explosion）**：深层链式法则连乘，某层梯度极大，传到参数后一步跨太远，loss 瞬间 NaN。
- **loss spike**：训练中偶发的异常 batch / 数据分布漂移 / 不稳定数值，使某 step 梯度激增。
- **RLHF/PPO 的 reward 波动**：reward 量级不稳，policy 梯度方差大，易爆炸。

裁剪的思路：**给梯度设上限**，超过就缩到上限内，让单步更新不失控。它不解决根本不稳定（那是 lr/warmup/正则的事），但能"兜底"防灾难性发散。

## 3. 核心概念详解

### 3.1 按范数裁剪（clip by global norm，最常用）

计算**所有参数梯度的全局 L2 范数**：

$$
\|G\| = \sqrt{\sum_p \sum_i g_{p,i}^2}
$$

若 $\|G\| > \tau$（阈值），整体缩放：

$$
g \leftarrow g\cdot\frac{\tau}{\max(\|G\|,\tau)} = g\cdot\min\!\left(1,\frac{\tau}{\|G\|}\right)
$$

否则不动。特点：**只缩不放**，且**保持梯度方向不变**（仅乘标量），不改变相对结构。

- PyTorch：`torch.nn.utils.clip_grad_norm_(params, max_norm=1.0)`。
- GPT/Llama 系列 `max_norm=1.0` 是标配。

### 3.2 按值裁剪（clip by value）

逐元素截断到 $[-c,c]$：

$$
g_i \leftarrow \max(-c,\min(g_i,c))
$$

特点：**改变方向**（大分量被截，小分量不动），可能扭曲梯度。RNN/某些早期训练用，LLM 少用。

- PyTorch：`clip_grad_value_(params, clip_value=0.5)`。

### 3.3 两者对比

| 项 | clip by norm | clip by value |
|---|---|---|
| 改方向 | ❌（只缩放） | ✅（截断） |
| 阈值语义 | 整体梯度范数上限 | 单元素上下限 |
| LLM 主流 | ✅ | ❌ |
| 调参难度 | 中（要估合理范数） | 低（直观） |

> [!tip] 为什么 LLM 选 norm
> norm 裁剪保留方向，只限制"步长总量"，符合"防爆但不扭曲更新"的目标；value 裁剪会扭曲稀疏大梯度（如 embedding），不友好。

### 3.4 阈值怎么定

- 经验：`max_norm=1.0`（GPT/Llama 系）。
- 原则：阈值应略大于**正常训练梯度范数的典型量级**，只在异常时触发，平时不动。可用日志统计梯度范数分布来定。

## 4. 数学原理 / 公式

### 全局范数为何有效

梯度下降单步更新量为 $\Delta\theta=-\eta g$，其范数 $\|\Delta\theta\|=\eta\|g\|$。clip 把 $\|g\|$ 限制在 $\tau$ 内，于是 $\|\Delta\theta\|\le\eta\tau$ —— **单步参数移动有上界**，从几何上保证不会一步跨太远而越出"可信一阶近似"区域。

### 与 lr 的等价性

clip 后等效 lr $\eta_{\text{eff}}=\eta\cdot\min(1,\tau/\|g\|)$。梯度大时自动降 lr，是一种"自适应降 lr"的简易机制（但只降不升）。

### 不解决小梯度

clip 只对大梯度生效，小梯度不动 → 不影响正常收敛，只兜底防爆。

## 5. 代码示例

```python
import torch
import torch.nn as nn

model = nn.Linear(100, 2)
opt = torch.optim.AdamW(model.parameters(), lr=3e-4)
x, y = torch.randn(32, 100), torch.randn(32, 2)
loss_fn = nn.MSELoss()

for step in range(5):
    opt.zero_grad(set_to_none=True)
    loss = loss_fn(model(x), y)
    loss.backward()

    # 关键：backward 之后、step 之前裁剪
    grad_norm = torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
    # 返回裁剪前的全局范数，用于监控
    print(f"step {step} pre-clip grad_norm={grad_norm.item():.4f}")

    opt.step()

# 监控梯度范数判断是否常触发
# 若 grad_norm 经常远超 1.0 → 说明训练不稳，要查 lr/warmup/数据，而不是光靠 clip 兜底

# 按值裁剪（不推荐 LLM 用）
# torch.nn.utils.clip_grad_value_(model.parameters(), clip_value=0.5)
```

## 6. 与其他知识点的关系

- **上游（依赖）**: [[backward过程]]（提供 `.grad`）、[[SGD]]、[[Adam与AdamW]]（step 前裁剪）。
- **下游（应用）**: 训练稳定性（防 spike/NaN）、[[PPO]]（clip policy 梯度防 KL 爆炸）、[[gradient explosion]]（RNN/深网络）。
- **对比 / 易混**:
  - **梯度裁剪 vs PPO 的 clipped objective**：前者裁**梯度值**（优化器层，防爆）；后者裁**目标函数**里 ratio（损失层，防 policy 跳太远）。两者名字像但作用层不同，PPO 两者都用。
  - **clip norm vs clip value**：见 3.3。
  - **clip vs 降 lr**：clip 是"异常时降"，降 lr 是"全程降"；clip 是兜底，调 lr 是治本。

## 7. 常见误区与易错点

> [!warning] 误区清单
> 1. **裁剪位置错**：必须在 `backward()` 后、`step()` 前；放反了要么裁到旧梯度、要么没裁到。
> 2. **把 clip 当万能稳定剂**：clip 只兜底防爆，治标不治本；频繁触发说明 lr/warmup/数据有问题，要去查。
> 3. **阈值设太小**：平时就触发，等效一直降 lr，收敛变慢。
> 4. **阈值设太大**：形同虚设，异常时不触发，照样爆炸。
> 5. **混淆 PPO 的两个 clip**：policy 梯度 clip（防爆）≠ PPO clipped objective（防 ratio 越界），别混。
> 6. **clip 后不监控**：不看 grad_norm 就不知道是否触发、训练是否健康。
> 7. **clip value 扭曲稀疏梯度**：LLM embedding 大梯度被截断方向，不如 norm 友好。

## 8. 延伸细节

### 8.1 与 warmup 的协同

warmup 前期小 lr + clip 双保险，是 LLM 防前期 spike 的标配。warmup 结束后 clip 仍保留作兜底。

### 8.2 梯度范数监控

健康训练的 grad_norm 应平稳、偶尔小峰值；若频繁尖峰或单调上升，提示不稳定（数据/分布/数值），要排查而非依赖 clip。

### 8.3 PPO 里的特殊用法

PPO 的 policy 更新对梯度极敏感（KL 易爆），常用更激进的 clip（如 max_norm=0.5 或 1.0）+ 小 lr + KL penalty 多管齐下，见 [[PPO]]、[[KL penalty]]。

### 8.4 bf16 下的注意

bf16 训练时，clip 前的 grad_norm 计算用 fp32 累加（防 bf16 精度丢），否则范数估错、clip 失准。fused optimizer 内部已处理。

### 8.5 与 gradient checkpointing 的关系

[[gradient-checkpointing]] 重算前向时梯度照常算，clip 流程不受影响，但要确保重算的中间值精度足够。
