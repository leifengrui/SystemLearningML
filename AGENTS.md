# AGENTS.md — 系统学习ML 工作规则

> **每次会话开始，必须先完整阅读本文件，再执行任何任务。**

本仓库是一个 **Obsidian 知识库**，用于系统学习机器学习 / 深度学习 / LLM / 强化学习 / 训推系统。
Obsidian vault 根目录为 `systemLearning/`，主索引起点为 `systemLearning/整体目录.md`。其各个笔记中'【user】符号表示我的批注 你需要解释、回答、保存相应的回答，必要时可以新建文件并且新建索引'

> **自动执行（重要）**：每次会话开始、完整读完本文件后，须**立即**按第九节流程，用检索工具扫描全库 `【user】` 并逐条回答、保存，**无需用户额外指令**。每完成一次笔记编辑后也顺手复查，防止新增未答批注。

---

## 一、核心原则

1. **直接给出知识，不要外引课程**
   - 禁止"建议你去学 CS231n / 动手学深度学习 / 某某章节"这类话术。
   - 用户要的是**当场把知识点讲透**，不是被指到一个外部链接再去啃。
   - 所有概念、公式、推导、代码，直接在笔记里写完整。

2. **不要让用户跳来跳去**
   - 一个知识点展开后，尽量在该文件内讲清楚，不要"详见 xxx 课程""请参考第几章"。
   - 跨知识点关联用 Obsidian 双链 `[[]]` 标注即可，但当前条目的核心内容必须自包含。

3. **内容要详实**
   - 不要只给一句话定义就收工。每个知识点要讲清：是什么、为什么、数学原理、代码、误区、关联。
   - 宁可写长，不要写空。

4. **Obsidian 格式优先**
   - 知识点之间用 `[[文件名]]` 双链互链，形成图谱。
   - 适当使用 callout (`> [!note]`, `> [!warning]`, `> [!tip]`)、数学公式 (`$...$` / `$$...$$`)、代码块、表格。
   - 文件名用中文，与目录条目一致，便于检索与双链。

## 三、统一笔记模板

**每个知识点文件都必须遵循以下模板**（小节标题固定，不适用的小节写"本条不涉及"而非删除，保持检索一致性）：

```markdown
# {知识点名称}

> **所属章节**: [[小节名]]
> **所属模块**: [[章名]]


## 1. 一句话定义
{用最朴素的语言说清这是什么}

## 2. 为什么需要它（动机与背景）
{它解决了什么问题，没有它会怎样}

## 3. 核心概念详解
{逐点展开，可分小标题 ###}

## 4. 数学原理 / 公式
{推导、公式，用 $$ 块}

## 5. 代码示例（可选
{最小可运行片段，带注释说明每行意图}

## 6. 与其他知识点的关系
- **上游（依赖）**: [[xxx]]
- **下游（应用）**: [[xxx]]
- **对比 / 易混**: [[xxx]]

## 7. 常见误区与易错点
{列出典型错误理解}

## 8. 延伸细节
{更深的内容、工程实践、边界情况}

---
相关: [[索引]]
```

## 四、工作流程（每次展开知识点时）

1. **读 `整体目录.md`**，定位用户要展开的章节 / 小节 / 条目。
2. **确定目标文件路径**：`systemLearning/{章文件夹}/{小节文件夹}/{知识点}.md`。
3. 若文件夹不存在，先创建。
4. **按模板写入完整内容**，做到自包含、详实、可直接读懂。
5. 在 `整体目录.md` 对应条目处，把纯文本替换为 `[[]]` 双链（如 `Tensor` → `[[Tensor]]`），保持索引可点击。
6. 若新知识点与其他已有笔记有关联，在双方"与其他知识点的关系"小节互相补双链，形成双向图。

## 五、格式与命名规范

| 项目 | 规范 |
|---|---|
| 文件名 | 中文，与目录条目一致，不加序号前缀（例 `autograd机制.md`） |
| 标题 | 文件首行 `# {同名标题}` |
| 双链 | `[[文件名]]`（不含 `.md` 后缀） |
| 公式 | 行内 `$x$`，独立 `$$\n...\n$$` |
| 代码块 | 标注语言 ` ```python ` |
| Callout | `> [!warning] 误区` / `> [!tip] 实践` / `> [!note] 补充` |
| 表格 | 用 Markdown 表格做对比（如 SGD vs Adam） |
| 分隔 | 文件末尾用 `---` 再放 `相关: [[]]` |

## 六、内容质量要求

- **公式必推导**：不只甩公式，要讲每一步为什么。
- **代码必可跑**：片段要能粘贴运行，变量有定义，不省略关键行。
- **对比必列表**：相似概念（如 DPO vs PPO、DP vs DDP vs FSDP）用表格横向对比。
- **误区必列**：初学者常踩的坑要明示。
- **不杜撰**：不确定的数字 / 结论要标注"待核实"，不硬编。不确定的内容使用联网搜索skill

---

## 七、禁止事项

- ❌ "建议学习 xxx 课程 / 读 xxx 书的第 n 章"
- ❌ "详见某链接"然后把核心内容丢给外部资源
- ❌ 只写一句话定义就结束
- ❌ 不按模板乱写结构，导致无法统一检索
- ❌ 在笔记里留 TODO 空洞而不当场补全

---

## 八、检索友好性

- 每个文件顶部 frontmatter 区（引用块）写明所属章节与模块，方便 Obsidian 搜索时快速定位。
- 关键术语在文中**加粗**，便于 Ctrl+F / Obsidian 搜索。
- 同一概念若有多个叫法（如 "KV cache" = "键值缓存"），在"一句话定义"里并列写出，提升命中率。

## 九、批注（【user】）自动检索与回答机制

> **本节是硬性流程，每次会话启动读完本文件后立即执行，不等用户催。**

知识库笔记中以 `【user】` 开头的行是用户的即时批注 / 提问。agent 必须主动发现、当场回答、把答案**落盘**回文件，而不是只在对话里复述。本节固化该流程。

### 9.1 触发时机

- **会话启动**：开头读完 `AGENTS.md` 后，立刻进入 9.2 检索。
- **每次写完 / 改完笔记后**：顺手再检索一次，防止本次编辑引入新的未答批注。
- **用户显式要求时**（如"回答所有 【user】"）：立即执行。

### 9.2 检索方法

用 **Grep 工具**在整个 vault 内搜索正则 `【user】`：

- 范围：`systemLearning/`（**不含 `AGENTS.md` 本身**——里面的 `【user】` 是规则说明，不是批注）。
- 命中的每一处 = 一条待办批注，记录其 `文件路径:行号` 与批注正文。
- 逐条处理，全部清零才算完成。

### 9.3 逐条处理流程

对每条批注：

1. **定位上下文**：Read 该文件，读批注所在段落上下文（前后若干行 / 整节），搞清它在问什么、隶属哪个知识点。
2. **判断处理类型**：
   - **行内提问型**（如"randn 是什么""为什么要记 device"）→ 就地回答。
   - **新建笔记型**（批注含"单独展开 / 单独成篇 / 新建 / 更新目录"等字样）→ 新建文件 + 更新 `整体目录.md` 双链。
3. **回答并保存**：
   - **行内型**：**统一用 `> [!note] 解答：…` callout 写完整答案**（这是标准回答格式，便于视觉区分与日后检索，不要用裸段落）。callout 标题固定以"解答："开头；正文遵循第 4 步质量要求（公式推导、代码可跑、对比列表、误区、双链）。**替换掉原 `【user】` 行**，不留空标记、不留 TODO、不留 `【user】`。
   - **新建型**：按第三节模板在新路径 `systemLearning/{章}/{小节}/{知识点}.md` 建文件；在原批注位置用 `> [!note] 已展开为独立笔记` callout 指向 `[[新笔记名]]`；在 `整体目录.md` 对应小节追加 `[[新笔记名]]` 双链条目。
4. **质量要求**：答案须遵循第六节——公式必推导、代码必可跑、对比必列表、误区必列、不杜撰；**宁长勿空**；相关概念补 `[[双链]]`。
5. **双向链接**：新答案若涉及已有笔记，在双方"与其他知识点的关系"小节互补双链，形成图谱。
6. **归档到 QA.md**：每答完一条，立即在 `systemLearning/QA.md` 追加一条索引（格式见 9.4），不要最后批量补，防遗漏。

### 9.4 QA.md 索引格式与维护

`systemLearning/QA.md` 是**批注问答索引**，把每条已回答的 `【user】` 登记成可检索条目，便于日后复盘复用。

- **位置**：`systemLearning/QA.md`（与 `整体目录.md` 同级）。
- **条目格式**：`- [[答案所在笔记名]] — {批注问题原句或精炼版}`，用 Obsidian 双链 `[[]]` 指向答案落盘的那篇笔记，点进去即可看答案。
- **分组**：按"章 / 小节"分二级标题组织，与 `整体目录.md` 结构对齐，便于浏览。
- **新建型批注**：索引指向**新建的那篇笔记**（即答案真正所在），而非批注原文件。
- **去重**：同一问题在多处重复出现，合并为一条并在后注出现位置。
- **维护时机**：每答完一条就追加，紧跟 9.3 第 6 步。

示例：

````markdown
# QA 索引

> 所有 `【user】` 批注的问题与答案落点。双链点进去即看答案。

## 一、ML/深度学习基础 / 张量与自动微分

- [[Tensor]] — tensor 怎么做到自动微分 / 记录梯度？
- [[数值类型与精度]] — 数值类型应单独展开（新建笔记）
- [[autograd机制]] — 梯度就是导数吗？方向最快变化？
````

### 9.5 完成校验

全部处理完后，**再用 Grep 搜一次 `【user】`**（范围 `systemLearning/`）：

- **无命中** → 全部已答，向用户报告清单（`文件:批注:处理方式`），并确认 `QA.md` 已同步登记。
- **仍有命中** → 继续处理，直到清零。

### 9.6 禁止

- ❌ 把 `【user】` 行留着不答、只口头复述。
- ❌ 答案只写一两句敷衍（违反"内容要详实"）。
- ❌ 新建笔记不更新 `整体目录.md`、不补双链，形成孤岛。
- ❌ 把答案写在对话里却**不落盘**到文件（用户要的是"保存"）。
- ❌ 检索后假装没看见、跳过某条。
- ❌ 答完批注却不登记到 `QA.md`，导致问答无法回溯检索。
```
