# 系统学习 ML · Obsidian 知识库
this md is for human. agents please read AGENTS.md
一个**自包含、可检索、双链成图**的机器学习 / 深度学习 / LLM / 强化学习 / 训推系统知识库，基于 [Obsidian](https://obsidian.md/) vault 组织。

不指向外部课程，不甩链接——所有知识点都在笔记里**当场讲透**：定义、动机、数学推导、可跑代码、误区、关联，一站读完。

---

## 一、这是什么

把 ML 全栈（从 Tensor / autograd 到 RLHF / 推理系统 / 分布式训练 / Ray 调度 / 训练稳定性）拆成**一个个独立知识点**，每个知识点按统一模板写成一篇笔记，笔记之间用 `[[双链]]` 互链，形成可点击的知识图谱。

- **不是**：课程笔记堆、链接合集、一句话定义清单。
- **是**：可直接 Ctrl+F / Obsidian 搜索查到、点开就读得懂、读完能上手的工程级参考库。

## 三、人类读者怎么用

### 1. 用 Obsidian 打开

1. 安装 [Obsidian](https://obsidian.md/)（免费）。
2. 打开 vault：选择 `系统学习ML/systemLearning/` 文件夹作为 vault 根目录。
3. 入口：打开 `整体目录.md`，这是全库主索引，所有章节、小节、知识点都是可点击的 `[[双链]]`。

> 不用 Obsidian 也能看——任意 Markdown 编辑器 / GitHub 网页渲染都能读，只是双链不会高亮跳转。

### 2. 学习路径建议

| 你现在的状态 | 推荐起点 |
|---|---|
| 刚入门 ML | 第一章 `01-ML深度学习基础/`（Tensor → autograd → 优化器 → 训练工程） |
| 懂 DL，要搞 LLM | 第二章 `02-Transformer与LLM核心结构/`（自注意力 → RoPE → KV cache → MoE） |
| 懂 LLM，要懂对齐 | 第五章 `05-LLM-RL对齐/`（SFT → RM → PPO → DPO） |
| 懂训练，要懂系统 | 第六~十一章（训推系统 / 分布式 / 推理系统 / Ray / 性能 / 显存） |
| 工程稳定性问题 | 第十二章 `12-训练稳定性工程/`（loss spike / 梯度爆炸 / warmup） |

### 3. 检索技巧

- **Obsidian 搜索**：`Ctrl/Cmd + O` 快速跳转任意笔记；`Ctrl/Cmd + F` 全库搜。
- **加粗术语**：每篇笔记的关键术语都**加粗**，便于搜中。
- **QA.md**：所有 `【user】` 批注的问题与答案落点都在 `systemLearning/QA.md` 索引里，点双链直达答案。

## 四、每篇笔记长什么样

统一 8 小节模板，便于检索与对照：

```markdown
# {知识点名称}

> **所属章节**: [[小节名]]
> **所属模块**: [[章名]]

## 1. 一句话定义
## 2. 为什么需要它（动机与背景）
## 3. 核心概念详解
## 4. 数学原理 / 公式       ← 公式必推导
## 5. 代码示例（可选）       ← 代码必可跑
## 6. 与其他知识点的关系     ← 上游/下游/对比 双链
## 7. 常见误区与易错点
## 8. 延伸细节

---
相关: [[索引]]
```

质量要求（见 `AGENTS.md` 第六节）：公式必推导、代码必可跑、对比必列表、误区必列、不杜撰。

## 五、AI agent 怎么用

本仓库支持用 AI 编程 agent（如 Claude Code / opencode / Cursor）持续维护与扩展。

### 1. 必读规则

- **`AGENTS.md`**：日常维护的唯一规则文档。agent 每次会话开始必须先完整读完，再执行任务。涵盖：
  - 笔记模板、命名规范、双链规则、质量要求。
  - **`【user】` 批注自动回答机制**：用户在笔记里用 `【user】` 开头的行写即时提问，agent 必须主动 grep 全库、逐条回答、答案用 `> [!note] 解答：…` callout 落盘回原文件、并登记到 `QA.md`。
- **`agents_join.md`**：一次性合并任务规则（把外部素材库查漏补缺并入本库），合并完成后作存档，日常维护不参考。

### 2. 让 agent 干活

把 agent 的工作目录设为 `系统学习ML/`，然后直接下指令：

- "展开第三章 DDP 这块" → agent 会读 `整体目录.md` 定位、按模板写笔记、回填双链。
- "回答所有 【user】" → agent 自动 grep、逐条答、落盘、登记 QA.md。
- "对比 DPO / PPO / IPO" → agent 在对应笔记补对比表 + 互链。

agent 的产出会自动遵循模板与质量要求，无需手动校正格式。

## 六、`【user】` 批注机制（人机协作核心）

这是本仓库**区别于普通笔记库**的关键设计：

1. 你在任意笔记里写一行 `【user】randn 和 rand 有什么区别？`
2. 下次 agent 启动会话时，自动 grep 全库发现这条批注。
3. agent 当场把答案写成 `> [!note] 解答：…` callout，**替换掉原 `【user】` 行**，落盘到原文件。
4. 同时在 `QA.md` 追加一条索引 `- [[笔记名]] — randn 和 rand 的区别`。
5. 校验：全库再 grep 一次 `【user】`，无命中才算完成。

效果：**你只管在笔记里随手提问，agent 会在下次会话自动消化并把答案永久存档**，不用专门开对话框问。

> [!warning] 注意
> `AGENTS.md` 本身里的 `【user】` 字样是**规则说明**，不是批注，agent 不会去答它。批注只算 `systemLearning/` 目录下的。

## 七、双链图谱

所有知识点用 Obsidian `[[文件名]]` 双链互链，无 `.md` 后缀。打开 Obsidian 的 **Graph View**（`Ctrl/Cmd + G`）可看到全库知识图谱。

例：`[[KV cache]]` 同时被 `[[自注意力]]`（上游）、`[[continuous batching]]`（应用）、`[[PD分离]]`（相关）引用，点进去在「6. 与其他知识点的关系」小节能看到完整上下游。

新笔记建成后会与相关已有笔记**双向补链**，避免孤岛。

## 八、贡献与扩展

如果你想往知识库加内容：

### 人类手动加
1. 确定知识点属于哪一章 / 小节（查 `整体目录.md`）。
2. 在对应文件夹按模板建 `.md`（文件名中文，与目录条目一致）。
3. 在 `整体目录.md` 对应条目把纯文本名改成 `[[笔记名]]` 双链。
4. 在相关已有笔记的「6. 与其他知识点的关系」补双链。

### 让 agent 加
直接对 agent 说"展开 XX"，它会自动完成上述全部步骤 + 质量校验。

## 九、不做什么

- ❌ 不整篇复制外部课程笔记（保持自包含、可独立读懂）。
- ❌ 不用 `[[外部文件]]` 双链指向库内不存在的文件（会成红链）。
- ❌ 不留 TODO 空洞——写到就写完。
- ❌ 不外引"建议去学 CS231n 第 N 章"——要讲就在笔记里讲透。
